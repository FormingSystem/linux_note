## 3.1_从模块开始_驱动的入口

### 3.1.1_为什么要有_模块

想象一下：内核是一座庞大的工厂，它本身已经能运转机器，但不同客户可能会带来新的需求（比如：要支持一块新的传感器，要驱动一块新的显卡）。
 如果每次都要把整个工厂拆掉重建（重编译内核），那成本太高。

于是，Linux 提供了**模块机制**：

- 就像插拔零件一样，内核允许你在运行时 **插入新的功能**（加载驱动模块）；
- 也允许你在不需要时 **拔掉它**（卸载驱动模块）。

------

### 3.1.2_一个最小的模块示例

```c
#include <linux/module.h>   // 模块相关宏与函数
#include <linux/init.h>     // __init 和 __exit 宏

/*
 * 模块加载时执行的函数
 * 相当于“驱动的入口”
 */
static int __init hello_init(void)
{
    printk(KERN_INFO "Hello, Linux 6.1 kernel!\n");
    return 0;   // 返回 0 表示加载成功
}

/*
 * 模块卸载时执行的函数
 * 相当于“驱动的出口”
 */
static void __exit hello_exit(void)
{
    printk(KERN_INFO "Goodbye, Linux 6.1 kernel!\n");
}

/* 指定入口和出口函数 */
module_init(hello_init);
module_exit(hello_exit);

/* 模块元信息 */
MODULE_LICENSE("GPL");                 // 遵循 GPL，避免内核“污染”警告
MODULE_AUTHOR("YourName");             // 作者
MODULE_DESCRIPTION("A simple hello");  // 简单描述
```

------

### 3.1.3_逐行理解

- `#include <linux/module.h>`
   模块的“必备头文件”，提供了 `module_init`、`module_exit` 等宏。
- `static int __init hello_init(void)`
   `__init` 表示：函数只在初始化阶段使用，执行完可以释放这段内存。
   加载模块时，内核会自动调用这个函数。
- `printk(KERN_INFO "...")`
   内核版本的 `printf`，日志等级 `KERN_INFO` 表示“普通信息”。
   信息会出现在 `dmesg`。
- `static void __exit hello_exit(void)`
   `__exit` 表示：只有卸载时才用到。
   卸载模块时，内核会自动调用它。
- `module_init` / `module_exit`
   分别告诉内核：入口函数是谁，出口函数是谁。
- `MODULE_LICENSE("GPL")`
   如果不写，加载时会提示 **“tainted kernel”**，说明模块许可证不明确。

------

### 3.1.4_编译与测试

1. 编写 `Makefile`：

```makefile
obj-m += hello.o

all:
	# 使用当前内核的构建系统来编译模块
	make -C /lib/modules/$(shell uname -r)/build M=$(PWD) modules

clean:
	make -C /lib/modules/$(shell uname -r)/build M=$(PWD) clean
```

1. 编译：

```bash
make
```

生成 `hello.ko`。

1. 加载与卸载：

```bash
sudo insmod hello.ko
dmesg | tail   # 可以看到 "Hello, Linux 6.1 kernel!"

sudo rmmod hello
dmesg | tail   # 可以看到 "Goodbye, Linux 6.1 kernel!"
```

------

### 3.1.5_小结

- 模块是**驱动开发的容器**；
- 加载时执行 `module_init` 指定的函数，卸载时执行 `module_exit` 指定的函数；
- `printk` 是调试的好帮手；
- 模块就像“插拔式零件”，后面我们会往里面装上**字符设备驱动**的逻辑。

## 3.2_Linux_模块名唯一

- **模块名必须唯一**。

- 构建时，Kbuild 会把 `obj-m`（或者 `obj-$(CONFIG_xxx)`）里指定的 **目标文件名去掉 `.o`** 作为模块名。

  - 例子：`obj-m += foo.o` → 编译生成 `foo.ko`，模块名就是 `foo`。

- 如果你在同一个内核构建里写了：

  ```make
  obj-m += foo.o
  obj-m += foo.o
  ```

  其实最后只会生成一个 `foo.ko`，相当于重复声明。

- **真正的冲突**是在「不同源码目录里，不同 `.c` 文件编译出的 `.ko` 文件模块名却一样」。

  - 比如 `drivers/usb/foo.c` 和 `drivers/net/foo.c`，如果你都写 `obj-m += foo.o`，编译阶段会报错：

    ```
    make[2]: *** No rule to make target 'foo.o', needed by '__build'.  Stop.
    ```

    或者最后只留下一个，另一个被覆盖。

  - 内核不会“二选一”，而是在 **构建阶段就冲突**，不会生成两个同名模块。

- 运行时，如果你手动强行加载两个同名模块（比如 `insmod foo.ko` 两次，或者 `foo.ko` 和 `foo_copy.ko` 但内部模块名都是 `foo`），**第二个加载会直接失败**：

  ```
  insmod: ERROR: could not insert module foo.ko: File exists
  ```

------

#### (1)_总结

1. **构建阶段**：
   - `obj-m` / `obj-m +=` 里模块名（`.o` 去掉后缀）重复，会报错或被覆盖；不会出现两个同名 `.ko` 并存。
2. **运行阶段**：
   - 内核模块表里模块名必须唯一，重名模块无法同时加载。
3. **实践建议**：
   - 模块名最好和功能/驱动唯一对应（比如 `usb_foo`, `net_bar`），避免与内核已有模块撞名。
   - 如果一定要相同功能多个版本，可以通过 `#define KBUILD_MODNAME "xxx"` 或者修改 Makefile 用不同的 `.o` 名字来区分。

## 3.3_Linux_模块操作命令说明

### 3.3.1_加载模块

#### (1)_insmod

```bash
sudo insmod demo.ko
```

- **作用**：把一个指定路径的 `.ko` 模块文件加载进内核。
- **特点**：
  - 只加载你指定的模块文件，不会自动解决依赖。
  - 文件名必须带路径（相对或绝对）。
- **常用场景**：测试自己编译的 `.ko` 驱动。

#### (2)_modprobe

```bash
sudo modprobe demo
```

- **作用**：按模块名加载模块。
- **特点**：
  - 会自动查找 `/lib/modules/$(uname -r)/` 下的 `.ko` 文件。
  - 会自动加载依赖模块（根据 `modules.dep`）。
- **常用场景**：在系统正式环境里加载模块，更智能。

------

### 3.3.2_卸载模块

#### (1)_rmmod

```bash
sudo rmmod demo
```

- **作用**：卸载已加载的模块（按模块名，而不是文件名）。
- **限制**：
  - 模块如果正被使用（有引用计数），会卸载失败。
  - 强制卸载用 `rmmod -f`（需要内核启用 `CONFIG_MODULE_FORCE_UNLOAD`，风险大）。

#### (2)_modprobe_-r

```bash
sudo modprobe -r demo
```

- **作用**：卸载模块及其依赖模块。
- **区别**：比 `rmmod` 更安全，能处理依赖关系。

------

### 3.3.3_查看模块状态

#### (1)_lsmod

```bash
lsmod | grep demo
```

- **作用**：列出当前内核已加载模块。
- **输出字段**：
  - 模块名
  - 内存大小
  - 使用计数（被多少其他模块/应用引用）

#### (2)_/proc/modules

```bash
cat /proc/modules
```

- 和 `lsmod` 本质一样，`lsmod` 只是对 `/proc/modules` 的格式化输出。

------

### 3.3.4_模块信息

#### (1)_modinfo

```bash
modinfo demo.ko
```

- **作用**：显示模块的元信息。
- **常见字段**：
  - `filename`：模块文件路径
  - `license`：许可证
  - `description`：描述
  - `author`：作者
  - `alias`：模块别名（modprobe 会用来匹配设备）
  - `depends`：依赖模块
  - `vermagic`：编译内核版本信息（必须和运行内核兼容）

------

### 3.3.5_调试相关

#### (1)_dmesg

```bash
dmesg | tail
```

- **作用**：查看最近的内核日志，加载/卸载模块时会打印 `printk` 输出。
- **用途**：调试模块初始化和退出函数的执行情况。

------

### 3.3.6_实际操作流程举例

1. 编译模块：

   ```bash
   make
   ```

   得到 `demo.ko`。

2. 加载模块（测试版）：

   ```bash
   sudo insmod demo.ko
   dmesg | tail   # 看初始化输出
   ```

3. 确认模块已加载：

   ```bash
   lsmod | grep demo
   ```

4. 卸载模块：

   ```bash
   sudo rmmod demo
   dmesg | tail   # 看退出输出
   ```

5. 如果模块已安装到内核目录：

   ```bash
   sudo modprobe demo
   sudo modprobe -r demo
   ```

## 3.4_设备号机制(major/minor/dev_t)

### 3.4.1_为什么需要设备号

在用户空间，我们访问 `/dev/xxx` 文件，比如 `/dev/ttyS0`。
 但是内核并不会靠 **字符串名字** 去找驱动，而是靠 **设备号 (device number)**。

设备号告诉内核两件事：

- **major**：这个 `/dev/xxx` 属于哪个驱动程序（即哪套 `file_operations`）。
- **minor**：这个驱动下面的具体哪一个实例（比如第一块硬盘还是第二块）。

👉 设备号就像：

- **楼号（major）**：告诉快递员这栋楼是哪家公司。
- **门牌号（minor）**：告诉快递员送到哪一间办公室。

------

### 3.4.2_dev_t_类型与辅助宏

内核用 `dev_t` 类型（本质是无符号整数）表示设备号。
 拆分和组合设备号，要用一组宏。

| 宏 / 类型             | 头文件             | 作用                             | 示例                   |
| --------------------- | ------------------ | -------------------------------- | ---------------------- |
| `dev_t`               | `<linux/types.h>`  | 设备号类型（内部存 major+minor） | `dev_t dev;`           |
| `MKDEV(major, minor)` | `<linux/kdev_t.h>` | 把主/次设备号组合成 dev_t        | `dev = MKDEV(200, 3);` |
| `MAJOR(dev)`          | `<linux/kdev_t.h>` | 从 dev_t 提取主设备号            | `MAJOR(dev)`           |
| `MINOR(dev)`          | `<linux/kdev_t.h>` | 从 dev_t 提取次设备号            | `MINOR(dev)`           |

------

### 3.4.3_代码示例(带详细注释)

```c
#include <linux/module.h>
#include <linux/init.h>
#include <linux/kdev_t.h>   // MKDEV, MAJOR, MINOR

static int __init devnum_demo_init(void)
{
    dev_t dev;  // 定义一个设备号变量
    int major, minor;

    // 手工构造一个设备号：主设备号200，次设备号5
    dev = MKDEV(200, 5);

    // 分别取出主设备号和次设备号
    major = MAJOR(dev);
    minor = MINOR(dev);

    printk(KERN_INFO "Demo device number: dev=%u, major=%d, minor=%d\n",
           dev, major, minor);

    return 0;
}

static void __exit devnum_demo_exit(void)
{
    printk(KERN_INFO "Device number demo exit.\n");
}

module_init(devnum_demo_init);
module_exit(devnum_demo_exit);

MODULE_LICENSE("GPL");
MODULE_AUTHOR("YourName");
MODULE_DESCRIPTION("Demo: major/minor device number");
```

加载模块后，`dmesg` 输出类似：

```
Demo device number: dev=1285, major=200, minor=5
```

（1285 是 dev_t 的整数值，内核内部用位运算把 major 和 minor 编码到一起）

------

### 3.4.4_设备号的申请与释放

在真正写字符设备驱动时，我们不能随便挑一个 major（比如 200），因为可能会和系统已有设备冲突。
 所以内核提供了两种方式：**静态分配** 和 **动态分配**。

| 函数                                                         | 头文件         | 作用                                         | 返回值                |
| ------------------------------------------------------------ | -------------- | -------------------------------------------- | --------------------- |
| `register_chrdev_region(dev_t from, unsigned count, const char *name)` | `<linux/fs.h>` | 静态申请设备号，`from` 固定（可能冲突）      | 0 成功，负数失败      |
| `alloc_chrdev_region(dev_t *dev, unsigned baseminor, unsigned count, const char *name)` | `<linux/fs.h>` | 动态申请设备号，系统自动分配未被占用的 major | 0 成功，*dev 保存结果 |
| `unregister_chrdev_region(dev_t from, unsigned count)`       | `<linux/fs.h>` | 释放之前申请的设备号                         | 无                    |

------

### 3.4.5_动态申请设备号示例

```c
#include <linux/module.h>
#include <linux/init.h>
#include <linux/fs.h>       // alloc_chrdev_region, unregister_chrdev_region

static dev_t devnum;        // 保存申请到的设备号

static int __init devnum_alloc_init(void)
{
    int ret;

    // baseminor=0，count=1，系统自动分配一个主设备号
    ret = alloc_chrdev_region(&devnum, 0, 1, "mychardev");
    if (ret < 0) {
        printk(KERN_ERR "Failed to allocate device number\n");
        return ret;
    }

    printk(KERN_INFO "Allocated device number: major=%d minor=%d\n",
           MAJOR(devnum), MINOR(devnum));

    return 0;
}

static void __exit devnum_alloc_exit(void)
{
    unregister_chrdev_region(devnum, 1); // 释放设备号
    printk(KERN_INFO "Device number released.\n");
}

module_init(devnum_alloc_init);
module_exit(devnum_alloc_exit);

MODULE_LICENSE("GPL");
MODULE_AUTHOR("YourName");
MODULE_DESCRIPTION("Demo: alloc_chrdev_region");
```

加载后 `dmesg`：

```
Allocated device number: major=241 minor=0
```

这里系统自动给了一个未占用的主设备号（如 241）。

------

### 3.4.6_/proc/devices_与_/dev/xxx

- 内核会在 `/proc/devices` 中记录所有注册的字符设备，格式是 **major + name**。
- `/dev/xxx` 节点的名字可以随便起（通常用 `udev` 自动生成），但它内部绑定的 **设备号** 必须和驱动一致。

例如：

```bash
cat /proc/devices
# 241 mychardev
```

手工创建设备节点：

```bash
sudo mknod /dev/mychardev0 c 241 0
```

此时打开 `/dev/mychardev0`，内核会用 `(241,0)` 找到我们的驱动。

------

📖 **小结：**

- 内核靠 **设备号（major+minor）** 找驱动，而不是字符串名字。
- **动态分配 major** 是推荐做法，避免冲突。
- `/dev/xxx` 文件只是“外壳”，里面的设备号才是关键。

## 3.5_cdev_机制与_VFS_的连接

### 3.5.1_为什么需要_cdev

内核把用户空间对 `/dev/xxx` 的访问转化为 `open/read/write/ioctl` 等系统调用。
 但是，内核必须知道：**这个设备号对应的驱动函数在哪里？**

- 设备号 = 门牌号
- `cdev` = 房产登记本（告诉内核：这个设备号属于哪个 `file_operations`）
- `file_operations` = 驱动接口集合（真正的门和钥匙）

所以，cdev 是“设备号”与“驱动函数”的桥梁。

------

### 3.5.2_struct_cdev_结构体

内核定义在 `<linux/cdev.h>`：

```c
struct cdev {
    struct kobject kobj;              // 内核对象，用于 sysfs
    struct module *owner;             // 所属模块（THIS_MODULE）
    const struct file_operations *ops;// 驱动函数集
    dev_t dev;                        // 设备号
    unsigned int count;               // 管理的次设备数量
};
```

关键字段：

- **ops**：指向 `struct file_operations`，决定用户空间的 read/write 调用哪个函数。
- **dev/count**：对应的设备号范围。
- **owner**：防止模块被卸载时，cdev 还在使用。

#### (1)_owner成员_必须是_THIS_MODULE吗

**不一定“必须”写成 `THIS_MODULE`，但在正常驱动里就应该写 `THIS_MODULE`。**

------

##### 1)_为什么通常要写_THIS_MODULE

- `struct module *owner` 用来做 **模块引用计数**。

- 当用户空间打开了这个字符设备，对应的模块引用计数会自动加 1（防止被 `rmmod` 卸载）；关闭设备时引用计数减 1。

- 如果你把它设为 `THIS_MODULE`，内核就能正确给当前驱动模块计数。

- 这就是为什么我们常见的写法是：

  ```c
  static const struct file_operations my_fops = {
      .owner = THIS_MODULE,
      .open = my_open,
      ...
  };
  ```

------

#### (2)_如果不是_THIS_MODULE_会怎样

- **设为 NULL**：
  - 内核不会自动给模块加引用计数。
  - 这样会有风险：设备打开时用户还在使用，你却能成功 `rmmod`，导致内核崩溃（因为 ops 里函数指针指向已卸载的代码段）。
- **设为别的模块指针**：
  - 理论上可以，比如某个“代理模块”统一管理多个子设备时，你可以把 `owner` 指向那个代理模块。
  - 但这是非常少见的情况，常规驱动开发几乎不会这么做。

------

#### (3)_总结

- **常规驱动**：一定要写 `THIS_MODULE`，保证引用计数正确。
- **特殊场景**：你要承担风险或者明确知道自己在做什么（例如设备节点由一个内核内置模块统一代理），才会不是 `THIS_MODULE`。
- **千万不要留空 (NULL)**，那就是一个大坑。

------

### 3.5.3_相关函数表

| 函数                                                         | 头文件           | 作用                             | 返回值           |

| `cdev_init(struct cdev *cdev, const struct file_operations *fops)` | `<linux/cdev.h>` | 初始化 cdev，绑定驱动函数表      | 无               |

### 3.5.4_示例代码

下面代码演示：

- 动态申请设备号
- 初始化 `cdev` 并注册
- 卸载时清理

```c
#include <linux/module.h>
#include <linux/init.h>
#include <linux/fs.h>       // alloc_chrdev_region, unregister_chrdev_region
#include <linux/cdev.h>     // cdev_init, cdev_add, cdev_del

static dev_t devnum;        // 保存设备号
static struct cdev my_cdev; // cdev 结构体实例

/* 虚拟的 file_operations，仅作示例 */
static int my_open(struct inode *inode, struct file *file)
{
    printk(KERN_INFO "mychardev: open()\n");
    return 0;
}

static int my_release(struct inode *inode, struct file *file)
{
    printk(KERN_INFO "mychardev: release()\n");
    return 0;
}

static ssize_t my_read(struct file *file, char __user *buf,
                       size_t count, loff_t *ppos)
{
    printk(KERN_INFO "mychardev: read()\n");
    return 0; // 暂时返回0，表示读到文件结尾
}

static ssize_t my_write(struct file *file, const char __user *buf,
                        size_t count, loff_t *ppos)
{
    printk(KERN_INFO "mychardev: write()\n");
    return count; // 假装写成功
}

/* 文件操作集 */
static struct file_operations fops = {
    .owner   = THIS_MODULE,   // 防止模块卸载
    .open    = my_open,
    .release = my_release,
    .read    = my_read,
    .write   = my_write,
};

/* 模块入口 */
static int __init my_init(void)
{
    int ret;

    // 1) 动态申请设备号（major 自动分配）
    ret = alloc_chrdev_region(&devnum, 0, 1, "mychardev");
    if (ret < 0) {
        printk(KERN_ERR "Failed to allocate device number\n");
        return ret;
    }
    printk(KERN_INFO "Allocated device number: major=%d minor=%d\n",
           MAJOR(devnum), MINOR(devnum));

    // 2) 初始化 cdev
    cdev_init(&my_cdev, &fops);
    my_cdev.owner = THIS_MODULE;

    // 3) 注册到内核
    ret = cdev_add(&my_cdev, devnum, 1);
    if (ret < 0) {
        unregister_chrdev_region(devnum, 1);
        printk(KERN_ERR "Failed to add cdev\n");
        return ret;
    }

    printk(KERN_INFO "mychardev loaded successfully\n");
    return 0;
}

/* 模块出口 */
static void __exit my_exit(void)
{
    cdev_del(&my_cdev);                // 注销 cdev
    unregister_chrdev_region(devnum,1);// 释放设备号
    printk(KERN_INFO "mychardev unloaded\n");
}

module_init(my_init);
module_exit(my_exit);

MODULE_LICENSE("GPL");
MODULE_AUTHOR("YourName");
MODULE_DESCRIPTION("Demo: cdev mechanism");
```

## 3.6_Linux_内核日志工具详解

### 3.6.1_dmesg_是什么

- **全称**：diagnostic message
- **作用**：显示内核环形缓冲区（ring buffer）里的消息。
- **来源**：内核代码里的 `printk()`、`pr_info()`、`pr_err()` 等日志输出。
- **特点**：
  - 日志是内核级的，不同于用户态应用的标准输出。
  - 信息量包括：驱动初始化、设备识别、异常错误、内核警告等。
  - 对调试驱动、查看启动过程非常关键。

```bash
dmesg
```

## 3.7_file_operations_深入解析(基于_Linux_6.1)

### 3.7.1_file_operations_是什么

在内核源码里（`include/linux/fs.h`）定义：

```c
struct file_operations {
    struct module *owner;                     // 拥有者模块，防止卸载
    loff_t (*llseek) (struct file *, loff_t, int);
    ssize_t (*read) (struct file *, char __user *, size_t, loff_t *);
    ssize_t (*write) (struct file *, const char __user *, size_t, loff_t *);
    ssize_t (*read_iter) (struct kiocb *, struct iov_iter *);
    ssize_t (*write_iter) (struct kiocb *, struct iov_iter *);
    int (*mmap) (struct file *, struct vm_area_struct *);
    unsigned long (*unlocked_ioctl) (struct file *, unsigned int, unsigned long);
    long (*compat_ioctl) (struct file *, unsigned int, unsigned long);
    int (*open) (struct inode *, struct file *);
    int (*release) (struct inode *, struct file *);
    ...
};
```

它就像一张“功能表”，每个成员是一个**函数指针**，由驱动作者来填。
 用户态调用系统调用，VFS 会查表，把调用转交给对应函数。

------

### 3.7.2_常见成员及作用

| 成员              | 头文件           | 作用                                   | 用户态对应调用                |
| ----------------- | ---------------- | -------------------------------------- | ----------------------------- |
| `.open`           | `<linux/fs.h>`   | 打开设备时调用，通常做初始化、资源分配 | `open("/dev/xxx", O_RDWR)`    |
| `.release`        | `<linux/fs.h>`   | 关闭设备时调用，释放资源               | `close(fd)`                   |
| `.read`           | `<linux/fs.h>`   | 从设备读数据                           | `read(fd, buf, count)`        |
| `.write`          | `<linux/fs.h>`   | 向设备写数据                           | `write(fd, buf, count)`       |
| `.llseek`         | `<linux/fs.h>`   | 改变文件偏移量                         | `lseek(fd, offset, SEEK_SET)` |
| `.unlocked_ioctl` | `<linux/fs.h>`   | 设备控制命令                           | `ioctl(fd, cmd, arg)`         |
| `.mmap`           | `<linux/fs.h>`   | 内存映射，把设备内存映射到用户态       | `mmap()`                      |
| `.poll`           | `<linux/poll.h>` | IO 多路复用支持                        | `select()/poll()/epoll()`     |

> **注意**：
>
> - `owner = THIS_MODULE` 必须填，否则模块可能在被使用时卸载，导致崩溃。
> - `.read_iter` / `.write_iter` 是更现代的接口，和 `.read` / `.write` 作用类似，但用于异步/分散读写。

------

### 3.7.3_一个最小化的_fops_示例

```c
/* 驱动函数集 */
static int my_open(struct inode *inode, struct file *file)
{
    printk(KERN_INFO "mychardev: open()\n");
    return 0;
}

static int my_release(struct inode *inode, struct file *file)
{
    printk(KERN_INFO "mychardev: release()\n");
    return 0;
}

static ssize_t my_read(struct file *file, char __user *buf,
                       size_t count, loff_t *ppos)
{
    char msg[] = "Hello from kernel!\n";
    size_t len = strlen(msg);

    // 防止多次读出相同内容：只有第一次返回数据
    if (*ppos >= len) return 0;

    // 把数据从内核拷贝到用户空间
    if (copy_to_user(buf, msg, len))
        return -EFAULT;

    *ppos += len;  // 更新文件偏移
    return len;    // 返回实际读出的字节数
}

static ssize_t my_write(struct file *file, const char __user *buf,
                        size_t count, loff_t *ppos)
{
    char kbuf[64];  // 内核缓冲区

    if (count > sizeof(kbuf) - 1)
        count = sizeof(kbuf) - 1;

    // 从用户空间拷贝数据到内核缓冲区
    if (copy_from_user(kbuf, buf, count))
        return -EFAULT;

    kbuf[count] = '\0';
    printk(KERN_INFO "mychardev: received '%s'\n", kbuf);

    return count; // 返回写入的字节数
}

/* 填充 file_operations */
static struct file_operations fops = {
    .owner   = THIS_MODULE,
    .open    = my_open,
    .release = my_release,
    .read    = my_read,
    .write   = my_write,
};
```

## 3.8_ioctl(设备控制接口)

### 3.8.1_ioctl_是什么

- `ioctl` 全称 **I/O control**。
- 它是一种 **设备专用操作接口**，允许用户空间向内核驱动发送控制命令。
- 设计动机：有些操作不好用 `read/write` 表达，比如：
  - 调整串口波特率；
  - 获取网卡 MAC 地址；
  - 控制 LED 开/关。

所以，`ioctl` 就是给每个驱动定义“专用命令”的途径。

------

### 3.8.2_内核函数原型

在 Linux 6.1 内核中，推荐使用 **unlocked_ioctl**（因为它在 BKL 之后被简化）。

```c
long (*unlocked_ioctl) (struct file *filp,
                        unsigned int cmd,
                        unsigned long arg);
```

参数：

- `filp`：文件指针，指向 `/dev/xxx`。
- `cmd`：用户空间传入的命令号（32 位整数，通常用宏定义）。
- `arg`：命令参数（可以是整数，也可以是用户态指针）。

返回值：

- `0`：成功；
- 负数：错误码（如 `-EINVAL`、`-EFAULT`）。

------

### 3.8.3_命令号的构造

内核提供一套宏（`<linux/ioctl.h>`），把 **方向 + 类型 + 序号 + 数据大小** 编码到一个整数里。

| 宏                          | 作用         | 示例                           |
| --------------------------- | ------------ | ------------------------------ |
| `_IO(type, nr)`             | 无参数命令   | `_IO('L', 0)`                  |
| `_IOR(type, nr, datatype)`  | 从驱动读数据 | `_IOR('L', 1, int)`            |
| `_IOW(type, nr, datatype)`  | 向驱动写数据 | `_IOW('L', 2, int)`            |
| `_IOWR(type, nr, datatype)` | 双向读写     | `_IOWR('L', 3, struct mydata)` |

- `type`：一个字符，区分不同设备（比如 `'L'` 表示 LED）。
- `nr`：命令序号。
- `datatype`：传递数据的类型（内核用来计算大小）。

------

### 3.8.4_驱动端实现示例

```c
#include <linux/uaccess.h>   // copy_to_user, copy_from_user
#include <linux/ioctl.h>     // _IO, _IOR, _IOW, _IOWR

#define LED_IOC_MAGIC   'L'             // 魔数
#define LED_ON          _IO(LED_IOC_MAGIC, 0)
#define LED_OFF         _IO(LED_IOC_MAGIC, 1)
#define LED_SET_BRIGHT  _IOW(LED_IOC_MAGIC, 2, int)
#define LED_GET_STATUS  _IOR(LED_IOC_MAGIC, 3, int)

static int led_status = 0;  // 0=off, 1=on
static int led_brightness = 0; // 亮度等级

static long my_ioctl(struct file *filp, unsigned int cmd, unsigned long arg)
{
    int tmp;

    switch (cmd) {
    case LED_ON:
        led_status = 1;
        printk(KERN_INFO "LED turned ON\n");
        break;

    case LED_OFF:
        led_status = 0;
        printk(KERN_INFO "LED turned OFF\n");
        break;

    case LED_SET_BRIGHT:
        if (copy_from_user(&tmp, (int __user *)arg, sizeof(int)))
            return -EFAULT;
        led_brightness = tmp;
        printk(KERN_INFO "LED brightness set to %d\n", led_brightness);
        break;

    case LED_GET_STATUS:
        tmp = led_status;
        if (copy_to_user((int __user *)arg, &tmp, sizeof(int)))
            return -EFAULT;
        break;

    default:
        return -EINVAL;  // 不支持的命令
    }

    return 0;
}

/* 更新 fops */
static struct file_operations fops = {
    .owner          = THIS_MODULE,
    .open           = my_open,
    .release        = my_release,
    .read           = my_read,
    .write          = my_write,
    .unlocked_ioctl = my_ioctl,  // 挂接 ioctl
};
```

### 3.8.5_用户态调用示例

用户程序 `test_ioctl.c`：

```c
#include <stdio.h>
#include <fcntl.h>
#include <sys/ioctl.h>

#define LED_IOC_MAGIC   'L'
#define LED_ON          _IO(LED_IOC_MAGIC, 0)
#define LED_OFF         _IO(LED_IOC_MAGIC, 1)
#define LED_SET_BRIGHT  _IOW(LED_IOC_MAGIC, 2, int)
#define LED_GET_STATUS  _IOR(LED_IOC_MAGIC, 3, int)

int main(void)
{
    int fd, status, bright = 5;

    fd = open("/dev/mychardev0", O_RDWR);
    if (fd < 0) {
        perror("open");
        return -1;
    }

    ioctl(fd, LED_ON);                  // 开灯
    ioctl(fd, LED_SET_BRIGHT, &bright); // 设置亮度
    ioctl(fd, LED_GET_STATUS, &status); // 读取状态

    printf("LED status = %d\n", status);

    ioctl(fd, LED_OFF);                 // 关灯
    close(fd);
    return 0;
}
```

  ```

  LED brightness set to 5
  ```

- 用户态输出：

  ```

  ```

- 数据在用户态和内核态之间传递，要用 `copy_to_user` / `copy_from_user`。
- `unlocked_ioctl` 是推荐的入口，老的 `ioctl` 已废弃。

## 3.9_poll_机制(poll/select/epoll_与驱动)

### 3.9.1_背景_为什么需要_poll

在用户态：

- `read()` 是阻塞的，如果没有数据，调用会一直挂起。
- 如果应用程序需要同时监听多个设备（比如串口、管道、网络套接字），就不能单纯依赖 `read()`。

于是，Linux 提供了 **I/O 多路复用**机制：

- `poll()`
- `select()`
- `epoll()`（更高效的扩展）

👉 它们的本质：让内核告诉用户 **设备是否可读/可写/有异常**，从而避免忙轮询。

------

### 3.9.2_驱动层的接口_file_operations.poll

在内核 6.1 中，驱动可以实现：

```c
__poll_t (*poll) (struct file *filp, struct poll_table_struct *wait);
```

参数：

- `filp`：文件指针。
- `wait`：poll_table，用户态传入，用来建立等待队列。

返回值：

- 一组 **事件掩码**，常见宏定义在 `<linux/poll.h>`：

| 宏                       | 含义 | 用户态对应                |
| ------------------------ | ---- | ------------------------- |
| `POLLIN` / `POLLRDNORM`  | 可读 | `pollfd.events & POLLIN`  |
| `POLLOUT` / `POLLWRNORM` | 可写 | `pollfd.events & POLLOUT` |
| `POLLERR`                | 错误 | `pollfd.events & POLLERR` |
| `POLLHUP`                | 挂起 | `pollfd.events & POLLHUP` |

------

### 3.9.3_驱动端实现步骤

1. 定义一个 **等待队列**（`wait_queue_head_t`）。
2. 在 `.poll` 里调用 `poll_wait()` 把进程挂入等待队列。
3. 根据设备状态返回对应的事件掩码。
4. 当设备状态变化时，调用 `wake_up_interruptible()` 唤醒等待的进程。

------

### 3.9.4_示例驱动(模拟可读事件)

```c
#include <linux/poll.h>   // poll_wait, POLLIN

static wait_queue_head_t my_queue; // 等待队列
static int data_ready = 0;         // 模拟数据状态

static ssize_t my_write(struct file *file, const char __user *buf,
                        size_t count, loff_t *ppos)
{
    data_ready = 1;                       // 写操作后标记有数据
    wake_up_interruptible(&my_queue);     // 唤醒等待队列
    printk(KERN_INFO "Data written, wake up readers\n");
    return count;
}

static ssize_t my_read(struct file *file, char __user *buf,
                       size_t count, loff_t *ppos)
{
    if (data_ready == 0)
        return 0; // 没数据返回 EOF（简单处理）

    data_ready = 0; // 读一次就清空
    return 0;
}

static __poll_t my_poll(struct file *filp, struct poll_table_struct *wait)
{
    __poll_t mask = 0;

    // 建立等待关系：如果进程调用 poll/select，会加入 my_queue
    poll_wait(filp, &my_queue, wait);

    // 如果有数据可读，返回 POLLIN
    if (data_ready)
        mask |= POLLIN | POLLRDNORM;

    // 如果总是可写，可以返回 POLLOUT
    mask |= POLLOUT | POLLWRNORM;

    return mask;
}

/* 文件操作表 */
static struct file_operations fops = {
    .owner   = THIS_MODULE,
    .read    = my_read,
    .write   = my_write,
    .poll    = my_poll,
};
```

3. 用户程序输出：

   ```

   ```

### 3.9.6_小结

- `.poll` 让驱动支持 **I/O 多路复用**，是异步/非阻塞编程的核心。
- 驱动里要维护 **等待队列**，并在事件发生时调用 `wake_up_interruptible()`。
- 返回值是 **事件掩码**，用户态通过 `poll/select/epoll` 判断设备是否可读/可写。
- 常见应用：串口、管道、键盘输入、网络套接字、GPIO 驱动。

## 3.10_mmap(内存映射机制)

### 3.10.1_为什么需要_mmap

- `read/write` 每次都要 `copy_to_user` / `copy_from_user`，效率不高。
- 对于 **大数据量设备**（如显存、视频缓冲、环形缓冲区），频繁拷贝性能低下。
- `mmap` 允许用户空间直接访问内核的内存区域：
  - 用户空间调用 `mmap()`；
  - 内核驱动实现 `.mmap`，把物理页/内核缓冲区映射给用户；
  - 用户空间得到一个指针，可以像操作数组一样直接访问。

👉 这就是**零拷贝**（zero-copy）的思路。

------

### 3.10.2_file_operations.mmap_接口

原型（`include/linux/fs.h`）：

```c
int (*mmap) (struct file *filp, struct vm_area_struct *vma);
```

- `filp`：文件指针。
- `vma`：虚拟内存区域描述（用户态 `mmap()` 请求的区间）。
  - `vma->vm_start`：用户空间起始地址。
  - `vma->vm_end`：用户空间结束地址。
  - `vm_pgoff`：偏移量（以页为单位）。

驱动需要调用 **remap_pfn_range()**，把物理页框映射到用户空间。

------

### 3.10.3_内核函数表(mmap_相关)

| 函数                                           | 头文件          | 作用                                      |
| ---------------------------------------------- | --------------- | ----------------------------------------- |
| `remap_pfn_range(vma, start, pfn, size, prot)` | `<linux/mm.h>`  | 把物理页帧号 (pfn) 映射到用户空间         |
| `vm_insert_page(vma, addr, page)`              | `<linux/mm.h>`  | 插入单页（可替代 remap_pfn_range 的场景） |
| `virt_to_phys(kaddr)`                          | `<asm/page.h>`  | 内核虚拟地址转物理地址                    |
| `__get_free_pages(flags, order)`               | `<linux/gfp.h>` | 分配连续物理页                            |
| `free_pages(addr, order)`                      | `<linux/gfp.h>` | 释放页                                    |

------

### 3.10.4_示例驱动(把内核缓冲区映射到用户空间)

下面代码演示：

- 驱动里分配一块内核内存；
- `.mmap` 里把这块内存映射到用户空间；
- 用户程序可以直接读写。

```c
#include <linux/mm.h>
#include <linux/slab.h>
#include <linux/fs.h>
#include <linux/cdev.h>
#include <linux/module.h>

#define BUF_SIZE (PAGE_SIZE)   // 映射 1 页

static dev_t devnum;
static struct cdev my_cdev;
static char *kernel_buf;       // 内核缓冲区（1 页）

/* mmap 回调函数 */
static int my_mmap(struct file *filp, struct vm_area_struct *vma)
{
    unsigned long pfn;

    // 获取物理页帧号
    pfn = virt_to_phys(kernel_buf) >> PAGE_SHIFT;

    // 将这段物理页映射到用户空间
    if (remap_pfn_range(vma,
                        vma->vm_start, // 用户空间起始地址
                        pfn,           // 物理页帧号
                        BUF_SIZE,      // 映射大小
                        vma->vm_page_prot)) // 访问权限
    {
        return -EAGAIN;
    }

    printk(KERN_INFO "mmap: mapped kernel_buf to user space\n");
    return 0;
}

/* fops */
static struct file_operations fops = {
    .owner = THIS_MODULE,
    .mmap  = my_mmap,
};

static int __init my_init(void)
{
    int ret;

    // 1) 动态申请设备号
    ret = alloc_chrdev_region(&devnum, 0, 1, "mmapdemo");
    if (ret < 0) return ret;

    // 2) 初始化 cdev
    cdev_init(&my_cdev, &fops);
    cdev_add(&my_cdev, devnum, 1);

    // 3) 分配一页内存
    kernel_buf = (char *)__get_free_page(GFP_KERNEL);
    if (!kernel_buf) {
        cdev_del(&my_cdev);
        unregister_chrdev_region(devnum, 1);
        return -ENOMEM;
    }

    strcpy(kernel_buf, "Hello from kernel mmap!\n");

    printk(KERN_INFO "mmapdemo loaded, major=%d\n", MAJOR(devnum));
    return 0;
}

static void __exit my_exit(void)
{
    free_page((unsigned long)kernel_buf); // 释放页
    cdev_del(&my_cdev);
    unregister_chrdev_region(devnum, 1);
    printk(KERN_INFO "mmapdemo unloaded\n");
}

module_init(my_init);
module_exit(my_exit);
MODULE_LICENSE("GPL");
```

### 3.10.5_用户态测试程序

```c
#include <stdio.h>
#include <fcntl.h>
#include <sys/mman.h>
#include <string.h>
#include <unistd.h>

#define BUF_SIZE 4096

int main()
{
    int fd = open("/dev/mmapdemo0", O_RDWR);
    if (fd < 0) { perror("open"); return -1; }

    // 映射一页内存
    char *addr = mmap(NULL, BUF_SIZE, PROT_READ | PROT_WRITE,
                      MAP_SHARED, fd, 0);
    if (addr == MAP_FAILED) { perror("mmap"); return -1; }

    printf("User read: %s\n", addr);

    // 写回数据
    strcpy(addr, "Message from user space!\n");

    munmap(addr, BUF_SIZE);
    close(fd);
    return 0;
}
```

运行效果：

- 用户程序输出：


- 如果用 `dmesg`，可以看到 `mmap: mapped kernel_buf to user space`。

- 内核缓冲区内容也被用户修改了（内核能看到 `"Message from user space!\n"`）。

### 3.10.6_小结

- `mmap` 提供**零拷贝**机制，适合大数据量设备（显存、DMA 缓冲）。
- 驱动通过 `remap_pfn_range` 把物理页映射到用户空间。
- 内存必须是 **页对齐** 的（通常用 `__get_free_page` 分配）。
- 用户态用 `mmap()` 获取指针，直接操作共享内存。

## 3.11_驱动源码(mychardev.c)

```c
// SPDX-License-Identifier: GPL-2.0
/*
 * mychardev.c
 *
 * 一个“从模块到字符设备”的完整示例（Linux 6.1 语义）
 * 特性：
 *   - 动态分配设备号（major 自动）
 *   - cdev + file_operations
 *   - 环形缓冲区（read/write）
 *   - ioctl 控制命令
 *   - poll/select/epoll 支持（可读事件）
 *   - mmap 把缓冲区直接映射到用户空间（零拷贝）
 *   - 自动创建设备节点 /dev/mychardev0
 *
 * 结构与并发说明（简化版）：
 *   - 使用自旋锁保护环形缓冲区的读写索引
 *   - 使用等待队列在写入后唤醒读者（配合 .poll）
 *   - 为演示简洁起见，没有加入阻塞 read 的等待逻辑（可自行扩展）
 */

#include <linux/module.h>
#include <linux/init.h>
#include <linux/fs.h>          // alloc_chrdev_region, unregister_chrdev_region, struct file_operations
#include <linux/cdev.h>        // cdev, cdev_init, cdev_add, cdev_del
#include <linux/device.h>      // class_create, device_create, class_destroy, device_destroy
#include <linux/uaccess.h>     // copy_to_user, copy_from_user
#include <linux/slab.h>        // kmalloc, kfree
#include <linux/spinlock.h>    // spinlock
#include <linux/poll.h>        // poll_wait, POLLIN, POLLOUT
#include <linux/mm.h>          // vm_area_struct, remap_pfn_range, vm_page_prot
#include <linux/ioctl.h>       // _IO, _IOR, _IOW, _IOWR
#include <linux/gfp.h>         // __get_free_pages, free_pages
#include <linux/version.h>

#define DEV_NAME        "mychardev"
#define CLASS_NAME      "mychardev"   // /sys/class/mychardev/...
#define DEV_COUNT       1

/* 环形缓冲区大小：整页对齐，方便 mmap */
#define RBUF_ORDER      0              // 2^0 页
#define RBUF_SIZE       (PAGE_SIZE << RBUF_ORDER)

/* ============ ioctl 命令定义（示例） ============ */
#define IOCTL_MAGIC     'M'
#define IOCTL_CLEAR     _IO(IOCTL_MAGIC, 0)               // 清空缓冲区
#define IOCTL_GET_SIZE  _IOR(IOCTL_MAGIC, 1, unsigned)    // 获取缓冲区大小
#define IOCTL_SET_MODE  _IOW(IOCTL_MAGIC, 2, int)         // 设置模式（自定义）

/* 设备对象 */
struct mychardev {
    dev_t               devno;          // 设备号(major:minor)
    struct cdev         cdev;           // cdev 实例
    struct class       *cls;            // sysfs class 对象
    struct device      *dev;            // /dev 节点的 device

    /* 环形缓冲区与并发控制 */
    char               *rbuf;           // 环形缓冲底层内存（整页，便于 mmap）
    size_t              rbuf_size;      // = RBUF_SIZE
    size_t              r_head;         // 写入位置（生产者）
    size_t              r_tail;         // 读取位置（消费者）
    spinlock_t          rlock;          // 保护 head/tail 的自旋锁

    /* poll 等待队列：写入后唤醒读者 */
    wait_queue_head_t   rd_queue;

    /* 演示参数 */
    int                 mode;           // IOCTL 可设置的“模式”
};

static struct mychardev gdev;

/* 工具：计算环形缓冲“当前可读/可写字节” */
static inline size_t rbuf_data_avail(struct mychardev *d)
{
    // head 写入位置，tail 读取位置
    // 已有数据 = head - tail (mod rbuf_size)
    return (d->r_head + d->rbuf_size - d->r_tail) % d->rbuf_size;
}
static inline size_t rbuf_space_avail(struct mychardev *d)
{
    // 可写空间 = rbuf_size - 1 - data_avail
    // 预留 1 字节区分满/空
    return d->rbuf_size - 1 - rbuf_data_avail(d);
}

/* ============ file_operations 回调 ============ */

/* open：打开放大器（这里不分配私有数据，简单记录） */
static int my_open(struct inode *inode, struct file *file)
{
    // 可以使用 container_of 从 inode->i_cdev 找回我们的 gdev，但我们只有一个设备，用全局即可
    try_module_get(THIS_MODULE); // 与 .owner=THIS_MODULE 作用相同，多一层保险
    pr_info(DEV_NAME ": open\n");
    return 0;
}

/* release：关闭设备（释放资源） */
static int my_release(struct inode *inode, struct file *file)
{
    module_put(THIS_MODULE);
    pr_info(DEV_NAME ": release\n");
    return 0;
}

/* read：从环形缓冲区读数据 */
static ssize_t my_read(struct file *file, char __user *buf,
                       size_t count, loff_t *ppos)
{
    size_t can_read, n1, n2;
    unsigned long flags;

    if (count == 0)
        return 0;

    spin_lock_irqsave(&gdev.rlock, flags);

    can_read = rbuf_data_avail(&gdev);
    if (can_read == 0) {
        // 简化处理：没有数据直接返回 0（EOF 语义）
        // 真实驱动可在这里 sleep，直到有数据（wait_event_interruptible）
        spin_unlock_irqrestore(&gdev.rlock, flags);
        return 0;
    }

    if (count > can_read)
        count = can_read;

    /* 计算分两段拷贝：从 tail 到缓冲末尾，再从头开始 */
    n1 = min(count, gdev.rbuf_size - gdev.r_tail);
    n2 = count - n1;

    spin_unlock_irqrestore(&gdev.rlock, flags);

    // 拷贝到用户
    if (copy_to_user(buf, gdev.rbuf + gdev.r_tail, n1))
        return -EFAULT;
    if (n2 && copy_to_user(buf + n1, gdev.rbuf, n2))
        return -EFAULT;

    // 更新 tail
    spin_lock_irqsave(&gdev.rlock, flags);
    gdev.r_tail = (gdev.r_tail + count) % gdev.rbuf_size;
    spin_unlock_irqrestore(&gdev.rlock, flags);

    return count;
}

/* write：写入环形缓冲区 */
static ssize_t my_write(struct file *file, const char __user *buf,
                        size_t count, loff_t *ppos)
{
    size_t space, n1, n2;
    unsigned long flags;

    if (count == 0)
        return 0;

    spin_lock_irqsave(&gdev.rlock, flags);
    space = rbuf_space_avail(&gdev);
    if (space == 0) {
        // 简化：满了直接丢弃（返回 0）
        // 可扩展为阻塞直到有空间
        spin_unlock_irqrestore(&gdev.rlock, flags);
        return 0;
    }
    if (count > space)
        count = space;

    n1 = min(count, gdev.rbuf_size - gdev.r_head);
    n2 = count - n1;
    spin_unlock_irqrestore(&gdev.rlock, flags);

    if (copy_from_user(gdev.rbuf + gdev.r_head, buf, n1))
        return -EFAULT;
    if (n2 && copy_from_user(gdev.rbuf, buf + n1, n2))
        return -EFAULT;

    spin_lock_irqsave(&gdev.rlock, flags);
    gdev.r_head = (gdev.r_head + count) % gdev.rbuf_size;
    spin_unlock_irqrestore(&gdev.rlock, flags);

    // 唤醒等待可读的进程（与 .poll 配合）
    wake_up_interruptible(&gdev.rd_queue);

    return count;
}

/* llseek：可选，实现文件偏移（本设备不使用，返回 -ESPIPE） */
static loff_t my_llseek(struct file *file, loff_t off, int whence)
{
    return -ESPIPE; // 非定位设备
}

/* ioctl：设备控制面板 */
static long my_ioctl(struct file *filp, unsigned int cmd, unsigned long arg)
{
    unsigned long flags;
    int tmp;

    switch (cmd) {
    case IOCTL_CLEAR:
        spin_lock_irqsave(&gdev.rlock, flags);
        gdev.r_head = gdev.r_tail = 0;
        spin_unlock_irqrestore(&gdev.rlock, flags);
        pr_info(DEV_NAME ": buffer cleared\n");
        return 0;

    case IOCTL_GET_SIZE:
        tmp = gdev.rbuf_size;
        if (copy_to_user((int __user *)arg, &tmp, sizeof(tmp)))
            return -EFAULT;
        return 0;

    case IOCTL_SET_MODE:
        if (copy_from_user(&tmp, (int __user *)arg, sizeof(tmp)))
            return -EFAULT;
        gdev.mode = tmp;
        pr_info(DEV_NAME ": mode set to %d\n", gdev.mode);
        return 0;

    default:
        return -EINVAL;
    }
}

/* poll：支持 poll/select/epoll */
static __poll_t my_poll(struct file *filp, struct poll_table_struct *wait)
{
    __poll_t mask = 0;
    unsigned long flags;

    // 将当前进程加入等待队列（当有数据写入时被唤醒）
    poll_wait(filp, &gdev.rd_queue, wait);

    spin_lock_irqsave(&gdev.rlock, flags);
    if (rbuf_data_avail(&gdev) > 0)
        mask |= POLLIN | POLLRDNORM;   // 可读
    // 这里简化：总是可写
    mask |= POLLOUT | POLLWRNORM;      // 可写
    spin_unlock_irqrestore(&gdev.rlock, flags);

    return mask;
}

/* mmap：把环形缓冲区映射到用户空间（零拷贝访问） */
static int my_mmap(struct file *filp, struct vm_area_struct *vma)
{
    unsigned long pfn = virt_to_phys(gdev.rbuf) >> PAGE_SHIFT;
    size_t size = vma->vm_end - vma->vm_start;

    if (size > gdev.rbuf_size)
        return -EINVAL;

    if (remap_pfn_range(vma, vma->vm_start, pfn, size, vma->vm_page_prot))
        return -EAGAIN;

    pr_info(DEV_NAME ": mmap ok, size=%zu\n", size);
    return 0;
}

/* file_operations 填表 */
static const struct file_operations my_fops = {
    .owner          = THIS_MODULE,     // 防止使用期间卸载
    .open           = my_open,
    .release        = my_release,
    .read           = my_read,
    .write          = my_write,
    .llseek         = my_llseek,
    .unlocked_ioctl = my_ioctl,
    .poll           = my_poll,
    .mmap           = my_mmap,
};

/* ============ 模块加载与卸载 ============ */

static int __init my_init(void)
{
    int ret;

    /* 1) 动态申请设备号 */
    ret = alloc_chrdev_region(&gdev.devno, 0, DEV_COUNT, DEV_NAME);
    if (ret) {
        pr_err(DEV_NAME ": alloc_chrdev_region failed: %d\n", ret);
        return ret;
    }
    pr_info(DEV_NAME ": allocated major=%d minor=%d\n",
            MAJOR(gdev.devno), MINOR(gdev.devno));

    /* 2) 初始化 cdev 并注册 */
    cdev_init(&gdev.cdev, &my_fops);
    gdev.cdev.owner = THIS_MODULE;
    ret = cdev_add(&gdev.cdev, gdev.devno, DEV_COUNT);
    if (ret) {
        pr_err(DEV_NAME ": cdev_add failed: %d\n", ret);
        unregister_chrdev_region(gdev.devno, DEV_COUNT);
        return ret;
    }

    /* 3) 创建 class 和 /dev 节点（需要 udev） */
    gdev.cls = class_create(THIS_MODULE, CLASS_NAME);
    if (IS_ERR(gdev.cls)) {
        ret = PTR_ERR(gdev.cls);
        pr_err(DEV_NAME ": class_create failed: %d\n", ret);
        cdev_del(&gdev.cdev);
        unregister_chrdev_region(gdev.devno, DEV_COUNT);
        return ret;
    }

    gdev.dev = device_create(gdev.cls, NULL, gdev.devno, NULL, DEV_NAME "0");
    if (IS_ERR(gdev.dev)) {
        ret = PTR_ERR(gdev.dev);
        pr_err(DEV_NAME ": device_create failed: %d\n", ret);
        class_destroy(gdev.cls);
        cdev_del(&gdev.cdev);
        unregister_chrdev_region(gdev.devno, DEV_COUNT);
        return ret;
    }

    /* 4) 分配环形缓冲区（整页对齐，便于 mmap） */
    gdev.rbuf_size = RBUF_SIZE;
    gdev.rbuf = (char *)__get_free_pages(GFP_KERNEL | __GFP_ZERO, RBUF_ORDER);
    if (!gdev.rbuf) {
        pr_err(DEV_NAME ": allocate ring buffer failed\n");
        device_destroy(gdev.cls, gdev.devno);
        class_destroy(gdev.cls);
        cdev_del(&gdev.cdev);
        unregister_chrdev_region(gdev.devno, DEV_COUNT);
        return -ENOMEM;
    }

    /* 5) 初始化并发原语与状态 */
    spin_lock_init(&gdev.rlock);
    init_waitqueue_head(&gdev.rd_queue);
    gdev.r_head = gdev.r_tail = 0;
    gdev.mode = 0;

    pr_info(DEV_NAME ": loaded ok (/dev/%s0)\n", DEV_NAME);
    return 0;
}

static void __exit my_exit(void)
{
    /* 与加载对称：先用户可见对象，后 cdev/设备号，最后内存 */
    if (gdev.rbuf)
        free_pages((unsigned long)gdev.rbuf, RBUF_ORDER);

    device_destroy(gdev.cls, gdev.devno);
    class_destroy(gdev.cls);

    cdev_del(&gdev.cdev);
    unregister_chrdev_region(gdev.devno, DEV_COUNT);

    pr_info(DEV_NAME ": unloaded\n");
}

module_init(my_init);
module_exit(my_exit);

MODULE_LICENSE("GPL");
MODULE_AUTHOR("you");
MODULE_DESCRIPTION("All-in-one char device demo: read/write/ioctl/poll/mmap");
```

## 3.17_内核驱动(阻塞_+_多实例版)_mychardev_block_multi.c

```c
// SPDX-License-Identifier: GPL-2.0
/*
 * mychardev_block_multi.c
 *
 * 特性：
 *  - 多实例字符设备：/dev/mychardev0 ... /dev/mychardev(N-1)
 *  - 阻塞式环形缓冲：读空 -> 读者睡眠；写满 -> 写者睡眠
 *  - poll/select/epoll：可读/可写事件
 *  - ioctl：清空缓冲、查询大小/剩余、设置模式
 *  - mmap：将环形缓冲整页映射到用户空间
 *  - 完整错误路径与资源回收
 *
 * 重要说明：
 *  - 读写路径可能睡眠，因此使用 mutex+waitqueue（而非 spinlock）
 *  - mmap 仍需页对齐分配
 */

#include <linux/module.h>
#include <linux/init.h>
#include <linux/fs.h>
#include <linux/cdev.h>
#include <linux/device.h>
#include <linux/uaccess.h>
#include <linux/slab.h>
#include <linux/mutex.h>
#include <linux/wait.h>
#include <linux/poll.h>
#include <linux/mm.h>
#include <linux/ioctl.h>
#include <linux/gfp.h>

#define DEV_NAME        "mychardev"
#define CLASS_NAME      "mychardev"
#define DEV_COUNT       2           /* === 多实例数量，按需调大 === */

/* 环形缓冲大小：2^order 页，对齐便于 mmap */
#define RBUF_ORDER      0
#define RBUF_SIZE       (PAGE_SIZE << RBUF_ORDER)

/* ioctl 命令 */
#define IOCTL_MAGIC       'M'
#define IOCTL_CLEAR       _IO(IOCTL_MAGIC,  0)
#define IOCTL_GET_SIZE    _IOR(IOCTL_MAGIC, 1, unsigned)   /* 总容量 */
#define IOCTL_GET_LEVEL   _IOR(IOCTL_MAGIC, 2, unsigned)   /* 当前有效数据量 */
#define IOCTL_SET_MODE    _IOW(IOCTL_MAGIC, 3, int)        /* 示例参数 */

struct mydev {
    /* 标识/注册对象 */
    dev_t          devno;
    struct cdev    cdev;
    struct device *dev;

    /* 缓冲与状态 */
    char          *rbuf;
    size_t         size;      /* = RBUF_SIZE */
    size_t         head;      /* 写指针 */
    size_t         tail;      /* 读指针 */
    int            mode;      /* 示例参数 */

    /* 并发与等待 */
    struct mutex   lock;      /* 保护 head/tail，允许睡眠 */
    wait_queue_head_t rdq;    /* 读等待队列（等可读） */
    wait_queue_head_t wrq;    /* 写等待队列（等可写） */
};

static dev_t g_base;
static struct class *g_cls;
static struct mydev *g_devs;  /* 数组 [DEV_COUNT] */

static inline size_t rbuf_level(struct mydev *d)
{
    /* 有效数据量 */
    return (d->head + d->size - d->tail) % d->size;
}
static inline size_t rbuf_space(struct mydev *d)
{
    /* 预留1字节区分满/空 */
    return d->size - 1 - rbuf_level(d);
}

/* ========== file_operations 回调 ========== */

static int my_open(struct inode *inode, struct file *file)
{
    /* 将设备对象指针放进 file->private_data，后续路径可直接取用 */
    struct mydev *d = container_of(inode->i_cdev, struct mydev, cdev);
    file->private_data = d;
    try_module_get(THIS_MODULE);
    return 0;
}

static int my_release(struct inode *inode, struct file *file)
{
    module_put(THIS_MODULE);
    return 0;
}

static ssize_t my_read(struct file *file, char __user *buf,
                       size_t count, loff_t *ppos)
{
    struct mydev *d = file->private_data;
    ssize_t ret = 0;

    if (count == 0) return 0;

    mutex_lock(&d->lock);

    /* 当没有数据时，若用户以阻塞方式打开，则睡眠等待；非阻塞则返回 -EAGAIN */
    while (rbuf_level(d) == 0) {
        mutex_unlock(&d->lock);
        if (file->f_flags & O_NONBLOCK) return -EAGAIN;
        if (wait_event_interruptible(d->rdq, rbuf_level(d) > 0))
            return -ERESTARTSYS;
        mutex_lock(&d->lock);
    }

    /* 真正可读的字节数不超过 count */
    count = min(count, rbuf_level(d));

    /* 分两段复制（tail->end，及头部） */
    {
        size_t n1 = min(count, d->size - d->tail);
        size_t n2 = count - n1;

        mutex_unlock(&d->lock);
        if (copy_to_user(buf, d->rbuf + d->tail, n1)) return -EFAULT;
        if (n2 && copy_to_user(buf + n1, d->rbuf, n2)) return -EFAULT;
        mutex_lock(&d->lock);

        d->tail = (d->tail + count) % d->size;
        ret = count;
    }

    /* 数据取走后，可能有空间了，唤醒写者 */
    wake_up_interruptible(&d->wrq);

    mutex_unlock(&d->lock);
    return ret;
}

static ssize_t my_write(struct file *file, const char __user *buf,
                        size_t count, loff_t *ppos)
{
    struct mydev *d = file->private_data;
    ssize_t ret = 0;

    if (count == 0) return 0;

    mutex_lock(&d->lock);

    /* 当没有空间可写时，阻塞或立即返回 */
    while (rbuf_space(d) == 0) {
        mutex_unlock(&d->lock);
        if (file->f_flags & O_NONBLOCK) return -EAGAIN;
        if (wait_event_interruptible(d->wrq, rbuf_space(d) > 0))
            return -ERESTARTSYS;
        mutex_lock(&d->lock);
    }

    count = min(count, rbuf_space(d));

    {
        size_t n1 = min(count, d->size - d->head);
        size_t n2 = count - n1;

        mutex_unlock(&d->lock);
        if (copy_from_user(d->rbuf + d->head, buf, n1)) return -EFAULT;
        if (n2 && copy_from_user(d->rbuf, buf + n1, n2)) return -EFAULT;
        mutex_lock(&d->lock);

        d->head = (d->head + count) % d->size;
        ret = count;
    }

    /* 写入后数据可读，唤醒读者 */
    wake_up_interruptible(&d->rdq);

    mutex_unlock(&d->lock);
    return ret;
}

static loff_t my_llseek(struct file *f, loff_t o, int w) { return -ESPIPE; }

static long my_ioctl(struct file *f, unsigned int cmd, unsigned long arg)
{
    struct mydev *d = f->private_data;
    int tmp;
    unsigned val;

    switch (cmd) {
    case IOCTL_CLEAR:
        mutex_lock(&d->lock);
        d->head = d->tail = 0;
        mutex_unlock(&d->lock);
        /* 清空后可写，唤醒写者 */
        wake_up_interruptible(&d->wrq);
        return 0;

    case IOCTL_GET_SIZE:
        val = d->size;
        if (copy_to_user((unsigned __user *)arg, &val, sizeof(val)))
            return -EFAULT;
        return 0;

    case IOCTL_GET_LEVEL:
        mutex_lock(&d->lock);
        val = rbuf_level(d);
        mutex_unlock(&d->lock);
        if (copy_to_user((unsigned __user *)arg, &val, sizeof(val)))
            return -EFAULT;
        return 0;

    case IOCTL_SET_MODE:
        if (copy_from_user(&tmp, (int __user *)arg, sizeof(tmp)))
            return -EFAULT;
        d->mode = tmp;
        return 0;

    default:
        return -EINVAL;
    }
}

static __poll_t my_poll(struct file *f, struct poll_table_struct *wait)
{
    struct mydev *d = f->private_data;
    __poll_t mask = 0;

    /* 将当前进程加入等待队列：这里关心可读与可写两类事件 */
    poll_wait(f, &d->rdq, wait);
    poll_wait(f, &d->wrq, wait);

    mutex_lock(&d->lock);
    if (rbuf_level(d) > 0)
        mask |= POLLIN  | POLLRDNORM;   /* 可读 */
    if (rbuf_space(d) > 0)
        mask |= POLLOUT | POLLWRNORM;   /* 可写 */
    mutex_unlock(&d->lock);

    return mask;
}

static int my_mmap(struct file *f, struct vm_area_struct *vma)
{
    struct mydev *d = f->private_data;
    size_t len = vma->vm_end - vma->vm_start;
    unsigned long pfn = virt_to_phys(d->rbuf) >> PAGE_SHIFT;

    if (len > d->size) return -EINVAL;

    if (remap_pfn_range(vma, vma->vm_start, pfn, len, vma->vm_page_prot))
        return -EAGAIN;

    return 0;
}

static const struct file_operations my_fops = {
    .owner          = THIS_MODULE,
    .open           = my_open,
    .release        = my_release,
    .read           = my_read,
    .write          = my_write,
    .llseek         = my_llseek,
    .unlocked_ioctl = my_ioctl,
    .poll           = my_poll,
    .mmap           = my_mmap,
};

/* ========== 模块加载/卸载与资源管理 ========== */

static int __init my_init(void)
{
    int i, ret;

    /* 1) 分配设备号区间：minor 从 0..DEV_COUNT-1 */
    ret = alloc_chrdev_region(&g_base, 0, DEV_COUNT, DEV_NAME);
    if (ret) return ret;

    /* 2) 建立 class，便于自动创建设备节点 */
    g_cls = class_create(THIS_MODULE, CLASS_NAME);
    if (IS_ERR(g_cls)) {
        ret = PTR_ERR(g_cls);
        unregister_chrdev_region(g_base, DEV_COUNT);
        return ret;
    }

    /* 3) 分配并初始化 N 个实例 */
    g_devs = kcalloc(DEV_COUNT, sizeof(*g_devs), GFP_KERNEL);
    if (!g_devs) {
        class_destroy(g_cls);
        unregister_chrdev_region(g_base, DEV_COUNT);
        return -ENOMEM;
    }

    for (i = 0; i < DEV_COUNT; ++i) {
        struct mydev *d = &g_devs[i];

        d->devno = MKDEV(MAJOR(g_base), MINOR(g_base) + i);

        /* cdev 注册 */
        cdev_init(&d->cdev, &my_fops);
        d->cdev.owner = THIS_MODULE;
        ret = cdev_add(&d->cdev, d->devno, 1);
        if (ret) { i--; goto fail_cdev; }

        /* /dev 节点 */
        d->dev = device_create(g_cls, NULL, d->devno, NULL, DEV_NAME "%d", i);
        if (IS_ERR(d->dev)) {
            ret = PTR_ERR(d->dev);
            cdev_del(&d->cdev);
            i--; goto fail_cdev;
        }

        /* 缓冲与并发对象 */
        d->size = RBUF_SIZE;
        d->rbuf = (char *)__get_free_pages(GFP_KERNEL | __GFP_ZERO, RBUF_ORDER);
        if (!d->rbuf) {
            device_destroy(g_cls, d->devno);
            cdev_del(&d->cdev);
            ret = -ENOMEM;
            i--; goto fail_cdev;
        }

        d->head = d->tail = 0;
        d->mode = 0;
        mutex_init(&d->lock);
        init_waitqueue_head(&d->rdq);
        init_waitqueue_head(&d->wrq);
    }

    pr_info(DEV_NAME ": loaded. major=%d, count=%d\n", MAJOR(g_base), DEV_COUNT);
    return 0;

fail_cdev:
    /* 回滚已经创建的实例 */
    for (; i >= 0; --i) {
        struct mydev *d = &g_devs[i];
        if (d->rbuf) free_pages((unsigned long)d->rbuf, RBUF_ORDER);
        if (d->dev)  device_destroy(g_cls, d->devno);
        if (d->cdev.owner) cdev_del(&d->cdev);
    }
    kfree(g_devs);
    class_destroy(g_cls);
    unregister_chrdev_region(g_base, DEV_COUNT);
    return ret;
}

static void __exit my_exit(void)
{
    int i;
    for (i = 0; i < DEV_COUNT; ++i) {
        struct mydev *d = &g_devs[i];

        device_destroy(g_cls, d->devno);
        cdev_del(&d->cdev);

    class_destroy(g_cls);
    unregister_chrdev_region(g_base, DEV_COUNT);
    pr_info(DEV_NAME ": unloaded\n");
}
```

## 3.21_使用/改造建议(工程化要点)

- **阻塞读/写行为**：
  - 读：无数据→阻塞；写入后 `wake_up_interruptible(&rdq)` 唤醒。
  - 写：无空间→阻塞；读出后 `wake_up_interruptible(&wrq)` 唤醒。
  - 非阻塞：`open` 时加 `O_NONBLOCK`，立即返回 `-EAGAIN`。
- **并发控制**：
  - 睡眠路径必须用 `mutex`（不是 spinlock）。
  - `wait_event_interruptible()` 里使用**条件表达式**，避免虚假唤醒。
- **多实例**：
  - `DEV_COUNT` 控制设备数量；每个实例单独的缓冲与等待队列，互不影响。
  - 注册区间：`alloc_chrdev_region(base, 0, DEV_COUNT, name)`。
- **mmap**：
  - 映射长度 ≤ 缓冲大小，且缓冲需页对齐（我们用 `__get_free_pages`）。
  - 如需安全只读映射，可检查 `vma->vm_flags` 并调整 `vm_page_prot`。
- **IOCTL 规范**：
  - 为不同设备/子系统选不同 magic 字符，避免冲突。
  - 命令和数据结构变动时要兼容老版本或提供新命令号。

```c
// SPDX-License-Identifier: GPL-2.0
/*
 * misc_echo.c —— 使用 miscdevice 的最小字符设备
 * 功能：open/read/write/ioctl（可按需添加）
 */
#include <linux/module.h>
#include <linux/init.h>
#include <linux/fs.h>            // struct file_operations
#include <linux/miscdevice.h>    // miscdevice, misc_register
#include <linux/uaccess.h>       // copy_to_user, copy_from_user
#include <linux/ioctl.h>         // _IO/IOR/IOW

#define DEV_NAME "misc_echo"

/* 简单 ioctl 定义 */
#define IOCTL_MAGIC  'E'
#define ECHO_CLEAR   _IO(IOCTL_MAGIC, 0)

static char kbuf[256];
static size_t klen;

static int echo_open(struct inode *inode, struct file *file)
{
    pr_info("%s: open\n", DEV_NAME);
    return 0;
}

static int echo_release(struct inode *inode, struct file *file)
{
    pr_info("%s: release\n", DEV_NAME);
    return 0;
}

static ssize_t echo_read(struct file *file, char __user *buf,
                         size_t count, loff_t *ppos)
{
    size_t n = min(count, klen - (size_t)*ppos);
    if (*ppos >= klen) return 0;    // EOF
    if (copy_to_user(buf, kbuf + *ppos, n)) return -EFAULT;
    *ppos += n;
    return n;
}

static ssize_t echo_write(struct file *file, const char __user *buf,
                          size_t count, loff_t *ppos)
{
    size_t n = min(count, sizeof(kbuf) - 1);
    if (copy_from_user(kbuf, buf, n)) return -EFAULT;
    kbuf[n] = 0;
    klen = n;
    return n;
}

static long echo_ioctl(struct file *f, unsigned int cmd, unsigned long arg)
{
    switch (cmd) {
    case ECHO_CLEAR:
        kbuf[0] = 0; klen = 0;
        return 0;
    default:
        return -EINVAL;
    }
}

static const struct file_operations echo_fops = {
    .owner          = THIS_MODULE,
    .open           = echo_open,
    .release        = echo_release,
    .read           = echo_read,
    .write          = echo_write,
    .unlocked_ioctl = echo_ioctl,
};

/* 关键：一个 struct miscdevice 即可描述整个字符设备 */
static struct miscdevice echo_miscdev = {
    .minor = MISC_DYNAMIC_MINOR,   // 动态分配次设备号
    .name  = DEV_NAME,             // 会出现在 /dev/misc_echo
    .fops  = &echo_fops,
};

static int __init echo_init(void)
{
    int ret = misc_register(&echo_miscdev);
    if (ret) {
        pr_err("%s: register failed: %d\n", DEV_NAME, ret);
        return ret;
    }
    pr_info("%s: registered at /dev/%s\n", DEV_NAME, DEV_NAME);
    return 0;
}

static void __exit echo_exit(void)
{
    misc_deregister(&echo_miscdev);
    pr_info("%s: deregistered\n", DEV_NAME);
}

module_init(echo_init);
module_exit(echo_exit);
MODULE_LICENSE("GPL");
MODULE_AUTHOR("you");
MODULE_DESCRIPTION("miscdevice echo demo");
```

- **使用**：编译出 `misc_echo.ko` → `insmod` 后会出现 `/dev/misc_echo`。

- `ioctl`：用户态 `ioctl(fd, ECHO_CLEAR)` 可清空缓存。

## 3.23_现代化_I/O_read_iter_/_write_iter

### 3.23.1_为什么用_iter_接口

- `read_iter(struct kiocb *, struct iov_iter *)`、`write_iter(...)`
- 更贴合 **分散/聚集 I/O**（多个 iovec 一次提交）、**异步 I/O**、**零拷贝** 场景；
- 与 `splice`/`sendfile` 等高效路径更好集成；
- 在 6.x 内核中更“现代”，有的子系统会优先走 iter 路径。

### 3.23.2_接口/头文件对照表

| 接口                                            | 头文件          | 作用                           |
| ----------------------------------------------- | --------------- | ------------------------------ |
| `read_iter(struct kiocb *, struct iov_iter *)`  | `<linux/fs.h>`  | 读路径（迭代器式）             |
| `write_iter(struct kiocb *, struct iov_iter *)` | `<linux/fs.h>`  | 写路径（迭代器式）             |
| `copy_to_iter()` / `copy_from_iter()`           | `<linux/uio.h>` | 从内核到 iter / 从 iter 到内核 |
| `iov_iter_count()`                              | `<linux/uio.h>` | iter 总字节数                  |
| `iov_iter_is_bvec()/ubuf()/xarray()`            | `<linux/uio.h>` | iter 底层类型判断（可选）      |

> 你可以同时提供 `.read/.write` 与 `.read_iter/.write_iter`；VFS 会优先使用 iter 版本（根据路径）。
>  没有 iter 时，VFS 会回落到 legacy read/write。

### 3.23.3_带注释示例(把我们_阻塞环形缓冲_的读写改成_iter_风格)

```c
#include <linux/uio.h>   // copy_{to,from}_iter, iov_iter helpers

/* 假设有 mydev、rbuf_level/space、等待队列与 mutex 同之前模板 */

static ssize_t my_read_iter(struct kiocb *iocb, struct iov_iter *to)
{
    struct file *file = iocb->ki_filp;
    struct mydev *d = file->private_data;
    size_t want = iov_iter_count(to);  // 用户这次总共想读多少字节
    ssize_t done = 0;

    if (!want) return 0;

    mutex_lock(&d->lock);
    while (want > 0) {
        size_t avail;

        /* 无数据：阻塞或返回 -EAGAIN */
        while ((avail = rbuf_level(d)) == 0) {
            if (file->f_flags & O_NONBLOCK) {
                mutex_unlock(&d->lock);
                return done ? done : -EAGAIN;
            }
            mutex_unlock(&d->lock);
            if (wait_event_interruptible(d->rdq, rbuf_level(d) > 0))
                return done ? done : -ERESTARTSYS;
            mutex_lock(&d->lock);
        }

        /* 实际本轮拷出的字节数 */
        {
            size_t n = min(want, avail);
            size_t n1 = min(n, d->size - d->tail);
            size_t n2 = n - n1;
            mutex_unlock(&d->lock);

            /* iter 风格：将内核数据拷到 iov_iter */
            if (n1) {
                size_t c = copy_to_iter(d->rbuf + d->tail, n1, to);
                if (c != n1) return done ? done : -EFAULT;
            }
            if (n2) {
                size_t c = copy_to_iter(d->rbuf, n2, to);
                if (c != n2) return done ? done : -EFAULT;
            }

            mutex_lock(&d->lock);
            d->tail = (d->tail + n) % d->size;
            done += n;
            want -= n;

            /* 读走数据后，唤醒写端 */
            wake_up_interruptible(&d->wrq);
        }

        /* 这里选择“尽量读满”，若想行为更像传统 read，可 break; */
    }
    mutex_unlock(&d->lock);
    return done;
}

static ssize_t my_write_iter(struct kiocb *iocb, struct iov_iter *from)
{
    struct file *file = iocb->ki_filp;
    struct mydev *d = file->private_data;
    size_t want = iov_iter_count(from);
    ssize_t done = 0;

    if (!want) return 0;

    mutex_lock(&d->lock);
    while (want > 0) {
        size_t space;

        /* 无空间：阻塞或返回 -EAGAIN */
        while ((space = rbuf_space(d)) == 0) {
            if (file->f_flags & O_NONBLOCK) {
                mutex_unlock(&d->lock);
                return done ? done : -EAGAIN;
            }
            mutex_unlock(&d->lock);
            if (wait_event_interruptible(d->wrq, rbuf_space(d) > 0))
                return done ? done : -ERESTARTSYS;
            mutex_lock(&d->lock);
        }

        /* 实际本轮写入字节数 */
        {
            size_t n = min(want, space);
            size_t n1 = min(n, d->size - d->head);
            size_t n2 = n - n1;
            mutex_unlock(&d->lock);

            /* iter 风格：从 iov_iter 拷数据到内核 */
            if (n1) {
                size_t c = copy_from_iter(d->rbuf + d->head, n1, from);
                if (c != n1) return done ? done : -EFAULT;
            }
            if (n2) {
                size_t c = copy_from_iter(d->rbuf, n2, from);
                if (c != n2) return done ? done : -EFAULT;
            }

            mutex_lock(&d->lock);
            d->head = (d->head + n) % d->size;
            done += n;
            want -= n;

            /* 写入后可读，唤醒读端 */
            wake_up_interruptible(&d->rdq);
        }
    }
    mutex_unlock(&d->lock);
    return done;
}

/* fops 中挂接（可与 .read/.write 并存） */
static const struct file_operations my_fops_iter = {
    .owner      = THIS_MODULE,
    .open       = my_open,
    .release    = my_release,
    .read_iter  = my_read_iter,     // ★
    .write_iter = my_write_iter,    // ★
    .unlocked_ioctl = my_ioctl,
    .poll       = my_poll,
    .mmap       = my_mmap,
    .llseek     = no_llseek,
};
```

#### (1)_关键点

- **`iov_iter_count(to/from)`** 得到本次能传输的总长度；
- **`copy_to_iter`/`copy_from_iter`** 分两段把数据搬运（照样配合环形缓冲）；
- 行为策略可选：“尽量读满/写满”或“读/写一轮就返回”；
- iter 版本在高吞吐、零拷贝管线里更“对味”。

## 3.24_一_DMA_+_MMAP_的正确打开方式(6.1_友好)

### 3.24.1_为什么要用_dma_alloc_coherent()_/_dma_mmap_coherent()

- **物理连续**、**DMA 设备可访问**的内存，且**缓存一致性**由内核/平台保证（或按平台策略处理），这是流式 I/O 的黄金路径。
- 直接用 `__get_free_pages()` + `remap_pfn_range()` 虽然能映射，但**未必可被外设安全 DMA**（IOMMU、cache、non-coherent SoC 等场景容易踩坑）。
- `dma_alloc_coherent()` 结合 `dma_mmap_coherent()`，是 **官方推荐** 的“可 mmap 的 DMA 缓冲”组合。

> 你的驱动若挂在 **platform/PCIe** 设备上，`struct device *dev` 通过 probe 获取；若是纯教学 demo 也可以临时使用 `NULL`，但强烈建议使用真实 `dev`，以便 IOMMU 与一致性策略生效。

------

### 3.24.2_缓存一致性(coherent_vs_streaming)

- **coherent（一致性）缓冲**：`dma_alloc_coherent()` 返回的内存，对 CPU 与 DMA 设备“天然一致”（或由体系结构保证）。**不用手工做 cache flush/invalidate**。
- **streaming（非一致）映射**：`dma_map_single()` / `dma_map_page()` 这类用于一次性 DMA，传输前/后需要 `dma_sync_*` 同步 cache，适合临时 I/O，不适合直接 mmap 给用户空间长期共享。

> **结论**：要把 DMA 缓冲 mmap 给用户空间用，优先使用 **coherent** 分配（也就是本节示例）。

------

### 3.24.3_IOMMU_/_non-coherent_SoC_注意点

- 有 IOMMU：`dma_alloc_coherent()` 返回的是 **CPU 虚拟地址** + **DMA 设备地址（dma_handle）**；映射到设备时用 `dma_handle`，映射到用户态用 `dma_mmap_coherent()`。
- 非一致性（如部分 ARMv8 平台）：coherent API 会按照平台策略处理一致性，不需要你手工 flush/invalidate。
- **禁止**把 `dma_handle` 当成 CPU 可解引用的指针；CPU 只能用返回的 `void *cpu_addr` 访问。

## 3.26_三_可运行示例_mydma_mmap.c(平台设备/教学风格)

> 说明：
>
> - 为了通用性，此处用“类 platform 设备”的写法（`THIS_MODULE` 级别），把 `dev` 简化为 **class 的 parent**（多数发行版可用）。在真实驱动里，请在 `probe()` 里拿到 **设备的 `struct device \*`**。
> - 代码包含：`open/read/write/ioctl/poll/mmap` 的最小组合；重点是 **DMA coherent + mmap**。
> - 用户态通过 `mmap` 得到一块能被设备 DMA 更新、CPU 直接读写的共享区。

```c
// SPDX-License-Identifier: GPL-2.0
/*
 * mydma_mmap.c
 * - 使用 dma_alloc_coherent() 分配 DMA 一致性缓冲
 * - 通过 dma_mmap_coherent() 将该缓冲 mmap 到用户空间
 * - 适用于 Linux 6.1 及同系
 */
#include <linux/module.h>
#include <linux/init.h>
#include <linux/fs.h>
#include <linux/cdev.h>
#include <linux/device.h>
#include <linux/uaccess.h>
#include <linux/poll.h>
#include <linux/mutex.h>
#include <linux/wait.h>
#include <linux/dma-mapping.h>  // dma_alloc_coherent, dma_mmap_coherent

#define DEV_NAME    "mydma"
#define CLASS_NAME  "mydma"
#define DEV_COUNT   1

/* 一页或多页都可以；真实场景按吞吐设置更大 */
#define DMA_ORDER   2
#define DMA_SIZE    (PAGE_SIZE << DMA_ORDER)

/* ioctl 示例 */
#include <linux/ioctl.h>
#define IOCTL_MAGIC      'D'
#define IOCTL_GET_DMASZ  _IOR(IOCTL_MAGIC, 1, unsigned)

/* 设备对象 */
struct mydma_dev {
    dev_t            devno;
    struct cdev      cdev;
    struct class    *cls;
    struct device   *dev;        // /dev 节点
    struct device   *parent;     // 用作 dma API 的 dev（教学演示用）
    /* DMA coherent 缓冲 */
    void            *cpu_addr;   // CPU 访问地址
    dma_addr_t       dma_handle; // 设备/DMA 访问地址
    size_t           size;

    /* 简单并发与事件 */
    struct mutex     lock;
    wait_queue_head_t rdq;
    int              readable;   // 写入一次 → 可读一次（示例语义）
};

static struct mydma_dev g;

/* ========== fops ========== */

static int my_open(struct inode *inode, struct file *filp)
{
    filp->private_data = &g;
    try_module_get(THIS_MODULE);
    return 0;
}
static int my_release(struct inode *inode, struct file *filp)
{
    module_put(THIS_MODULE);
    return 0;
}

/* 读：把 DMA 缓冲里的头一行返回（教学语义） */
static ssize_t my_read(struct file *f, char __user *buf, size_t cnt, loff_t *ppos)
{
    struct mydma_dev *d = f->private_data;
    ssize_t ret;
    size_t n;

    if (cnt == 0) return 0;

    /* 简易阻塞：没有可读事件时，阻塞或 -EAGAIN */
    if (!d->readable) {
        if (f->f_flags & O_NONBLOCK) return -EAGAIN;
        if (wait_event_interruptible(d->rdq, d->readable))
            return -ERESTARTSYS;
    }

    mutex_lock(&d->lock);
    /* 把缓冲区前 64 字节拷给用户（示例） */
    n = min(cnt, (size_t)64);
    if (copy_to_user(buf, d->cpu_addr, n)) {
        ret = -EFAULT;
    } else {
        d->readable = 0;     // 读消费一次
        ret = n;
    }
    mutex_unlock(&d->lock);
    return ret;
}

/* 写：把用户数据写入 DMA 缓冲区头部，唤醒读者（示例） */
static ssize_t my_write(struct file *f, const char __user *buf, size_t cnt, loff_t *ppos)
{
    struct mydma_dev *d = f->private_data;
    size_t n = min(cnt, d->size);
    ssize_t ret;

    mutex_lock(&d->lock);
    if (copy_from_user(d->cpu_addr, buf, n)) {
        ret = -EFAULT;
    } else {
        d->readable = 1;
        wake_up_interruptible(&d->rdq);
        ret = n;
    }
    mutex_unlock(&d->lock);
    return ret;
}

static __poll_t my_poll(struct file *f, struct poll_table_struct *wait)
{
    struct mydma_dev *d = f->private_data;
    __poll_t mask = POLLOUT | POLLWRNORM;  // 始终可写
    poll_wait(f, &d->rdq, wait);
    if (READ_ONCE(d->readable))
        mask |= POLLIN | POLLRDNORM;
    return mask;
}

static long my_ioctl(struct file *f, unsigned int cmd, unsigned long arg)
{
    struct mydma_dev *d = f->private_data;
    unsigned v;
    switch (cmd) {
    case IOCTL_GET_DMASZ:
        v = d->size;
        if (copy_to_user((unsigned __user *)arg, &v, sizeof(v)))
            return -EFAULT;
        return 0;
    default:
        return -EINVAL;
    }
}

/* mmap：关键点 —— 使用 dma_mmap_coherent() */
static int my_mmap(struct file *f, struct vm_area_struct *vma)
{
    struct mydma_dev *d = f->private_data;
    size_t len = vma->vm_end - vma->vm_start;
    if (len > d->size) return -EINVAL;

    /* 将 coherent DMA 缓冲映射到用户空间 */
    return dma_mmap_coherent(d->parent, vma, d->cpu_addr, d->dma_handle, d->size);
}

static const struct file_operations my_fops = {
    .owner          = THIS_MODULE,
    .open           = my_open,
    .release        = my_release,
    .read           = my_read,
    .write          = my_write,
    .poll           = my_poll,
    .unlocked_ioctl = my_ioctl,
    .mmap           = my_mmap,
    .llseek         = no_llseek,
};

/* ========== 模块装卸 ========== */

static int __init my_init(void)
{
    int ret;

    /* 1) 设备号 + cdev + /dev节点 */
    ret = alloc_chrdev_region(&g.devno, 0, DEV_COUNT, DEV_NAME);
    if (ret) return ret;

    cdev_init(&g.cdev, &my_fops);
    g.cdev.owner = THIS_MODULE;
    ret = cdev_add(&g.cdev, g.devno, DEV_COUNT);
    if (ret) goto err_chr;

    g.cls = class_create(THIS_MODULE, CLASS_NAME);
    if (IS_ERR(g.cls)) { ret = PTR_ERR(g.cls); goto err_cdev; }

    /* 用 class 的 kobject 作为 parent，便于 dma API 获取 dev。真实驱动请使用 probe 里的设备 dev。 */
    g.parent = &g.cls->p->subsys.kobj.parent->sd->s->kobj.sd->s->kobj.kset->kobj.parent ? NULL : NULL;
    /* ↑ 上一行仅为防止编译器告警而留的“空引用”；实际我们直接用 class 的 struct device 作为 parent */
    g.dev = device_create(g.cls, NULL, g.devno, NULL, DEV_NAME "0");
    if (IS_ERR(g.dev)) { ret = PTR_ERR(g.dev); goto err_cls; }
    /* 正确的 parent 应该是某个真实 device；这里教学用法：把 /sys/class/mydma/mydma0 对应 device 当 parent 使用 */
    g.parent = g.dev;

    /* 2) 分配 coherent DMA 缓冲 */
    g.size = DMA_SIZE;
    g.cpu_addr = dma_alloc_coherent(g.parent, g.size, &g.dma_handle, GFP_KERNEL);
    if (!g.cpu_addr) { ret = -ENOMEM; goto err_dev; }

    /* 3) 初始化并发与状态 */
    mutex_init(&g.lock);
    init_waitqueue_head(&g.rdq);
    g.readable = 0;

    pr_info("%s: loaded. major=%d, dma_size=%zu\n",
            DEV_NAME, MAJOR(g.devno), g.size);
    return 0;

err_dev:
    device_destroy(g.cls, g.devno);
err_cls:
    class_destroy(g.cls);
err_cdev:
    cdev_del(&g.cdev);
err_chr:
    unregister_chrdev_region(g.devno, DEV_COUNT);
    return ret;
}

static void __exit my_exit(void)
{
    if (g.cpu_addr)
        dma_free_coherent(g.parent, g.size, g.cpu_addr, g.dma_handle);

    device_destroy(g.cls, g.devno);
    class_destroy(g.cls);
    cdev_del(&g.cdev);
    unregister_chrdev_region(g.devno, DEV_COUNT);
    pr_info("%s: unloaded\n", DEV_NAME);
}

module_init(my_init);
module_exit(my_exit);
MODULE_LICENSE("GPL");
MODULE_AUTHOR("you");
MODULE_DESCRIPTION("DMA coherent + mmap demo (Linux 6.1)");
```

> 注：上面为了“纯内核模块教学”在没有真实 platform/PCIe 设备的情况下，把 `g.dev` 作为 `dma_alloc_coherent()` 的 `dev` 参数使用。真实项目中请在 `probe()` 得到 `struct device *` 后使用它。

## 3.28_五_常见踩坑与对策

1. **不要用 `remap_pfn_range()` 映射 coherent DMA 缓冲**
   - 它不会自动帮你处理缓存与设备权限；平台/IOMMU 下容易异常。
   - 正确做法：**`dma_mmap_coherent()`**。
2. **不要把 `dma_handle` 当作 CPU 指针**
   - `dma_handle` 是设备地址（或 IOVA），CPU 只能用 `cpu_addr` 访问。
3. **真实设备请在 `probe()` 上下文分配**
   - `dev` 必须是设备的 `struct device*`（PCIe 的 `&pdev->dev`、platform 的 `&pdev->dev`）。
   - 这样 IOMMU 映射、NUMA 亲和、缓存策略才是正确的。
4. **权限与安全**
   - 映射到用户空间即意味可被读写。必要时在 `.mmap` 中检查 `vma->vm_flags`（只读/可写），并使用 `pgprot_*` 调整。
   - 若只允许只读：拒绝 `PROT_WRITE` 或对 `vm_page_prot` 做保护（平台相关）。
5. **设备/应用同步**
   - 设备 DMA 写入与应用读取需要“生产者-消费者”协议（门铃寄存器、中断、head/tail 索引），否则数据边界不一致。
   - 本示例用 `readable` + `poll` 做了最简化演示；真实场景请配合中断/队列。
6. **大缓冲与内存压力**
   - `dma_alloc_coherent()` 可能分配失败（高阶页、内存碎片）。可尝试减小 `order` 或用 **CMA**/设备树保留内存。
   - 若必须巨量缓冲，可考虑 **videobuf2**、**dma-buf** 框架。

## 3.29_补充_如果你确实要用_streaming(dma_map_single())怎么办

- *不建议* 直接 mmap streaming 缓冲到用户态；但若必须：
  - 需要保证**固定 pin** 用户页（`pin_user_pages()`）+ 建立映射 + 正确的 cache sync（平台相关）+ 处理 IOMMU，工程复杂、坑多。
- 对于“用户态持续访问的共享区”，**coherent** 是正确选择。

## 3.30_架构概览

- **coherent DMA 区**：`dma_alloc_coherent()` 分配一整块；划分为 `N` 个固定大小的帧（`frame_size * frame_count`）。
- **元数据（内核）**：`head`（生产者写入后推进）、`tail`（消费者读出后推进）。
- **中断**：
  - 真实硬件：`request_threaded_irq()` → 顶半部（ack/最简）、线程化底半部（回收 DMA 描述符、更新 head、唤醒读者）。
  - 无硬件：`hrtimer` 周期性触发 → 调用同一个 bottom-half 处理函数（模拟“DMA 完成”）。
- **用户接口**：
  - `read()`：一次取一帧（拷贝到用户缓冲，适合简单脚本）。
  - `poll/epoll`：等待“有帧可读”。
  - `mmap()`：把整个帧环映射到用户态（零拷贝），用户根据 `ioctl` 得到的 `head/tail` 元数据做无锁消费。
  - `ioctl()`：查询 `frame_size`/`frame_count`、查询/复位 `head/tail`、启停“模拟 DMA”。

## 3.32_内核驱动(可直接编译运行)_mydma_irq_ring.c

> 特点：
>
> - coherent DMA 区 + 帧环；
> - **真实中断骨架** + **hrtimer 模拟**（默认启用模拟）；
> - `read()`：按帧输出；`poll`：就绪通知；`mmap()`：零拷贝；`ioctl()`：查询/控制。
> - 6.1 友好、全路径注释。

```c
// SPDX-License-Identifier: GPL-2.0
/*
 * mydma_irq_ring.c
 * DMA + IRQ + Ring buffer + poll + mmap (Linux 6.1)
 *
 * 架构：
 *  - coherent DMA 区分片为 frame ring；head/tail 为帧级索引
 *  - 真实硬件：request_threaded_irq() 顶半部最简 ack，线程化 bottom-half 更新 head
 *  - 无硬件：hrtimer 周期触发，调用与底半部相同的“生产者”逻辑，模拟 DMA 完成
 *  - 用户接口：read(按帧)、poll、mmap(零拷贝)、ioctl(查询/控制)
 */
#include <linux/module.h>
#include <linux/init.h>
#include <linux/fs.h>
#include <linux/cdev.h>
#include <linux/device.h>
#include <linux/uaccess.h>
#include <linux/mutex.h>
#include <linux/spinlock.h>
#include <linux/poll.h>
#include <linux/wait.h>
#include <linux/dma-mapping.h>
#include <linux/ioctl.h>
#include <linux/hrtimer.h>
#include <linux/ktime.h>
#include <linux/interrupt.h>

#define DEV_NAME     "mydmairq"
#define CLASS_NAME   "mydmairq"
#define DEV_COUNT    1

/* ---- Ring config: 可按需调整 ---- */
#define FRAME_SIZE        4096               /* 每帧大小（字节） */
#define FRAME_COUNT       64                 /* 帧个数（总大小 = size*count） */
#define DMA_BYTES         (FRAME_SIZE * FRAME_COUNT)

/* ---- 模拟中断开关（无硬件时启用） ---- */
#define USE_FAKE_IRQ      1
#define FAKE_IRQ_HZ       200                /* 每秒产生多少帧（模拟 DMA 完成率） */

/* ---- ioctl 定义 ---- */
#define IOCTL_MAGIC       'R'
#define IOCTL_GET_GEOM    _IOR(IOCTL_MAGIC, 1, struct ring_geom)  /* 查询帧几何 */
#define IOCTL_GET_INDEX   _IOR(IOCTL_MAGIC, 2, struct ring_index) /* 查询 head/tail */
#define IOCTL_CLR_INDEX   _IO(IOCTL_MAGIC,  3)                    /* 清空索引 */
#define IOCTL_RUN_FAKE    _IOW(IOCTL_MAGIC, 4, int)               /* 启停模拟DMA(1/0) */

struct ring_geom {
    unsigned frame_size;
    unsigned frame_count;
    unsigned total_size;
};

struct ring_index {
    unsigned head;    /* 生产者写入后的下一帧索引 */
    unsigned tail;    /* 消费者读走后的下一帧索引 */
};

/* 设备对象 */
struct mydev {
    /* 注册对象 */
    dev_t            devno;
    struct cdev      cdev;
    struct class    *cls;
    struct device   *dev;
    struct device   *parent;  /* 用作 dma API 的 device；真实项目中来自 probe */

    /* coherent DMA 缓冲（按帧环组织） */
    void            *cpu_addr;
    dma_addr_t       dma_handle;
    size_t           bytes;       /* = DMA_BYTES */

    /* 帧环索引（IRQ ctx 会更新 head） */
    unsigned         frame_size;
    unsigned         frame_count;
    unsigned         head;        /* 生产者推进 */
    unsigned         tail;        /* 消费者推进 */
    spinlock_t       idx_lock;    /* 保护 head/tail，需支持 IRQ ctx */

    /* 用户接口并发与事件 */
    wait_queue_head_t rdq;        /* 有帧可读时唤醒 */
    struct mutex     read_lock;   /* 串行化 read 的拷贝（非必须，示例友好） */

#if USE_FAKE_IRQ
    /* 软定时器模拟 DMA 完成中断 */
    struct hrtimer   timer;
    int              running;
    u64              seqno;       /* 生成的帧序号，用于示例数据 */
#endif

    /* 真实中断骨架（未接硬件时可不注册） */
    int              irq;         /* 硬件 IRQ 号（示例中不实际使用） */
};

static struct mydev g;

/* ========== 工具与 Ring 操作 ========== */
static inline void *frame_ptr(struct mydev *d, unsigned idx)
{
    return (char *)d->cpu_addr + (size_t)idx * d->frame_size;
}
static inline unsigned ring_next(struct mydev *d, unsigned i)
{
    return (i + 1) % d->frame_count;
}
static inline bool ring_full(struct mydev *d, unsigned head, unsigned tail)
{
    return ring_next(d, head) == tail;
}
static inline bool ring_empty(struct mydev *d, unsigned head, unsigned tail)
{
    return head == tail;
}

/* ========== “生产者”：在底半部/模拟中断中填充一帧并推进 head ========== */
/* 真实设备：这里应该是 DMA 完成后，根据 HW 报告得到哪一帧完成，然后更新 head */
static void produce_one_frame(struct mydev *d)
{
    unsigned long flags;
    unsigned head, tail;

    /* 抢占 head/tail 快照做判断 */
    spin_lock_irqsave(&d->idx_lock, flags);
    head = d->head;
    tail = d->tail;

    if (ring_full(d, head, tail)) {
        /* 溢出策略：丢弃最旧一帧（推进 tail），避免阻塞生产者。
         * 也可以选择丢弃新帧，或计数告警。 */
        d->tail = ring_next(d, tail);
    }

    /* 实际“硬件完成”的帧索引：就是当前 head */
    head = d->head;
    spin_unlock_irqrestore(&d->idx_lock, flags);

    /* === 填充帧内容（示例：首64字节写入序号/时间戳/签名）=== */
    {
        char *p = frame_ptr(d, head);
        memset(p, 0, d->frame_size);
#if USE_FAKE_IRQ
        snprintf(p, 64, "frame#%llu ts=%llu ns\n",
                 (unsigned long long)g.seqno++,
                 (unsigned long long)ktime_get_ns());
#else
        /* 真实设备：从 DMA 描述符/寄存器处获知帧信息，必要时 cache 同步 */
        snprintf(p, 64, "hw_frame ts=%llu ns\n",
                 (unsigned long long)ktime_get_ns());
#endif
    }

    /* 推进 head，并唤醒读者 */
    spin_lock_irqsave(&d->idx_lock, flags);
    d->head = ring_next(d, d->head);
    spin_unlock_irqrestore(&d->idx_lock, flags);

    wake_up_interruptible(&d->rdq);
}

/* ========== 真实中断骨架（可选） ========== */
static irqreturn_t my_irq_top(int irq, void *dev_id)
{
    /* 顶半部应尽量短：ack 中断、屏蔽或对 HW 做最小工作，然后唤醒 bottom-half */
    /* 示例：直接返回需要线程化处理 */
    return IRQ_WAKE_THREAD;
}

static irqreturn_t my_irq_thread(int irq, void *dev_id)
{
    struct mydev *d = dev_id;

    /* 真实设备中，这里应：
     *  - 读取 DMA 完成队列 / HW 状态寄存器
     *  - 回收/提交新的 DMA 描述符
     *  - 对每个完成的帧调用 produce_one_frame()
     */
    produce_one_frame(d);
    return IRQ_HANDLED;
}

/* ========== hrtimer 模拟“DMA 完成中断”（无硬件也能跑） ========== */
#if USE_FAKE_IRQ
static enum hrtimer_restart my_timer_cb(struct hrtimer *t)
{
    struct mydev *d = container_of(t, struct mydev, timer);
    if (READ_ONCE(d->running)) {
        produce_one_frame(d);
        /* 重新安排下一次“中断” */
        hrtimer_forward_now(&d->timer, ktime_set(0, NSEC_PER_SEC / FAKE_IRQ_HZ));
        return HRTIMER_RESTART;
    }
    return HRTIMER_NORESTART;
}
#endif

static int my_open(struct inode *inode, struct file *filp)
{
    filp->private_data = &g;
    try_module_get(THIS_MODULE);
    return 0;
}

static int my_release(struct inode *inode, struct file *filp)
{
    module_put(THIS_MODULE);
    return 0;

/* read：一次读一帧（拷贝方式）。更高效的路径请用 mmap + 用户态无拷贝访问。 */
static ssize_t my_read(struct file *f, char __user *buf, size_t cnt, loff_t *ppos)
{
    struct mydev *d = f->private_data;
    unsigned long flags;
    unsigned head, tail;
    ssize_t ret;

    if (cnt < d->frame_size)
        return -EINVAL;  /* 简化：要求调用方给足一帧空间 */

    /* 等待有帧可读（阻塞或非阻塞） */
    for (;;) {
        spin_lock_irqsave(&d->idx_lock, flags);
        head = d->head; tail = d->tail;
        if (!ring_empty(d, head, tail)) { /* 有帧 */
            spin_unlock_irqrestore(&d->idx_lock, flags);
            break;
        }
        spin_unlock_irqrestore(&d->idx_lock, flags);

        if (f->f_flags & O_NONBLOCK)
            return -EAGAIN;
        if (wait_event_interruptible(d->rdq, ({
                spin_lock_irqsave(&d->idx_lock, flags);
                head = d->head; tail = d->tail;
                bool has = !ring_empty(d, head, tail);
                spin_unlock_irqrestore(&d->idx_lock, flags);
                has;
            })))
            return -ERESTARTSYS;
    }

    /* 复制一帧 */
    mutex_lock(&d->read_lock);
    {
        void *src = frame_ptr(d, tail);
        if (copy_to_user(buf, src, d->frame_size)) {
            ret = -EFAULT;
            goto out_unlock;
        }
        ret = d->frame_size;

        /* 推进 tail */
        spin_lock_irqsave(&d->idx_lock, flags);
        d->tail = ring_next(d, d->tail);
        spin_unlock_irqrestore(&d->idx_lock, flags);
    }
out_unlock:
    mutex_unlock(&d->read_lock);
    return ret;
}

/* poll：有帧可读则返回 POLLIN */
static __poll_t my_poll(struct file *f, struct poll_table_struct *wait)
{
    struct mydev *d = f->private_data;
    unsigned long flags;
    __poll_t mask = POLLOUT | POLLWRNORM; /* 写路径这里不提供，标记可写仅示例 */

    poll_wait(f, &d->rdq, wait);

    spin_lock_irqsave(&d->idx_lock, flags);
    if (!ring_empty(d, d->head, d->tail))
        mask |= POLLIN | POLLRDNORM;
    spin_unlock_irqrestore(&d->idx_lock, flags);

    return mask;
}

/* mmap：把整个帧环映射给用户态（零拷贝） */
static int my_mmap(struct file *f, struct vm_area_struct *vma)
{
    struct mydev *d = f->private_data;
    size_t len = vma->vm_end - vma->vm_start;
    if (len > d->bytes) return -EINVAL;
    return dma_mmap_coherent(d->parent, vma, d->cpu_addr, d->dma_handle, d->bytes);
}

/* ioctl：查询几何/索引，清空索引，启停模拟中断 */
static long my_ioctl(struct file *f, unsigned int cmd, unsigned long arg)
{
    struct mydev *d = f->private_data;
    unsigned long flags;

    switch (cmd) {
    case IOCTL_GET_GEOM: {
        struct ring_geom g = {
            .frame_size  = d->frame_size,
            .frame_count = d->frame_count,
            .total_size  = d->bytes,
        };
        if (copy_to_user((void __user *)arg, &g, sizeof(g)))
            return -EFAULT;
        return 0;
    }
    case IOCTL_GET_INDEX: {
        struct ring_index i;
        spin_lock_irqsave(&d->idx_lock, flags);
        i.head = d->head; i.tail = d->tail;
        spin_unlock_irqrestore(&d->idx_lock, flags);
        if (copy_to_user((void __user *)arg, &i, sizeof(i)))
            return -EFAULT;
        return 0;
    }
    case IOCTL_CLR_INDEX: {
        spin_lock_irqsave(&d->idx_lock, flags);
        d->head = d->tail = 0;
        spin_unlock_irqrestore(&d->idx_lock, flags);
        return 0;
    }
#if USE_FAKE_IRQ
    case IOCTL_RUN_FAKE: {
        int on;
        if (copy_from_user(&on, (void __user *)arg, sizeof(on)))
            return -EFAULT;
        if (on) {
            if (!READ_ONCE(d->running)) {
                WRITE_ONCE(d->running, 1);
                hrtimer_start(&d->timer, ktime_set(0, NSEC_PER_SEC / FAKE_IRQ_HZ), HRTIMER_MODE_REL);
            }
        } else {
            WRITE_ONCE(d->running, 0);
            hrtimer_cancel(&d->timer);
        }
        return 0;
    }
#endif
    default:
        return -EINVAL;
    }
}

static const struct file_operations fops = {
    .owner          = THIS_MODULE,
    .open           = my_open,
    .release        = my_release,
    .read           = my_read,
    .poll           = my_poll,
    .mmap           = my_mmap,
    .unlocked_ioctl = my_ioctl,
    .llseek         = no_llseek,
};

/* ========== 模块装卸 ========== */

static int __init my_init(void)
{
    int ret;

    /* 1) 设备号 + cdev + /dev 节点 */
    ret = alloc_chrdev_region(&g.devno, 0, DEV_COUNT, DEV_NAME);
    if (ret) return ret;

    cdev_init(&g.cdev, &fops);
    g.cdev.owner = THIS_MODULE;
    ret = cdev_add(&g.cdev, g.devno, DEV_COUNT);
    if (ret) goto err_chr;

    g.cls = class_create(THIS_MODULE, CLASS_NAME);
    if (IS_ERR(g.cls)) { ret = PTR_ERR(g.cls); goto err_cdev; }

    g.dev = device_create(g.cls, NULL, g.devno, NULL, DEV_NAME "0");
    if (IS_ERR(g.dev)) { ret = PTR_ERR(g.dev); goto err_cls; }
    g.parent = g.dev;  /* 教学：拿这个 device 做 dma API 的 dev；真实项目请用 probe 提供的 dev */

    /* 2) coherent DMA 缓冲（帧环） */
    g.bytes       = DMA_BYTES;
    g.frame_size  = FRAME_SIZE;
    g.frame_count = FRAME_COUNT;
    g.cpu_addr = dma_alloc_coherent(g.parent, g.bytes, &g.dma_handle, GFP_KERNEL);
    if (!g.cpu_addr) { ret = -ENOMEM; goto err_dev; }
    memset(g.cpu_addr, 0, g.bytes);

    /* 3) 索引/并发与事件 */
    spin_lock_init(&g.idx_lock);
    init_waitqueue_head(&g.rdq);
    mutex_init(&g.read_lock);
    g.head = g.tail = 0;

#if USE_FAKE_IRQ
    /* 4) 启动“模拟中断” */
    hrtimer_init(&g.timer, CLOCK_MONOTONIC, HRTIMER_MODE_REL);
    g.timer.function = my_timer_cb;
    g.running = 1;
    g.seqno   = 0;
    hrtimer_start(&g.timer, ktime_set(0, NSEC_PER_SEC / FAKE_IRQ_HZ), HRTIMER_MODE_REL);
#else
    /* 4) 真实硬件中断注册（示意）
     * ret = request_threaded_irq(g.irq, my_irq_top, my_irq_thread, IRQF_SHARED, DEV_NAME, &g);
     * if (ret) goto err_dma;
     */
#endif

    pr_info("%s: loaded. major=%d frames=%u x %uB total=%zuB\n",
            DEV_NAME, MAJOR(g.devno), g.frame_count, g.frame_size, g.bytes);
    return 0;

#if !USE_FAKE_IRQ
err_dma:
#endif
    dma_free_coherent(g.parent, g.bytes, g.cpu_addr, g.dma_handle);
err_dev:
    device_destroy(g.cls, g.devno);
err_cls:
    class_destroy(g.cls);
err_cdev:
    cdev_del(&g.cdev);
err_chr:
    unregister_chrdev_region(g.devno, DEV_COUNT);
    return ret;
}
```

## 3.33_‍_用户态测试(轮询/epoll_+_mmap)test_ring.c

- 先用 `ioctl` 拿到几何参数；
- `mmap` 整个环；
- 再用 `poll` 等待可读；
- 通过 `IOCTL_GET_INDEX` 获取 `head/tail`，按“无锁协议”消费帧（**零拷贝**）；
- 也可以直接 `read()` 取帧（有拷贝，简单直观）。

```c
// gcc -O2 -Wall -o test_ring test_ring.c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <fcntl.h>
#include <sys/ioctl.h>
#include <sys/mman.h>
#include <sys/poll.h>
#include <unistd.h>
#include <errno.h>

#define DEV_PATH "/dev/mydmairq0"

#define IOCTL_MAGIC       'R'
struct ring_geom { unsigned frame_size, frame_count, total_size; };
struct ring_index { unsigned head, tail; };
#define IOCTL_GET_GEOM    _IOR(IOCTL_MAGIC, 1, struct ring_geom)
#define IOCTL_GET_INDEX   _IOR(IOCTL_MAGIC, 2, struct ring_index)
#define IOCTL_CLR_INDEX   _IO(IOCTL_MAGIC,  3)
#define IOCTL_RUN_FAKE    _IOW(IOCTL_MAGIC, 4, int)

static inline unsigned ring_next(unsigned i, unsigned n) { return (i+1)%n; }

int main(void)
{
    int fd = open(DEV_PATH, O_RDWR);
    if (fd < 0) { perror("open"); return 1; }

    /* geometry */
    struct ring_geom g;
    if (ioctl(fd, IOCTL_GET_GEOM, &g) < 0) { perror("ioctl geom"); return 1; }
    printf("ring: %u frames x %uB = %uB\n", g.frame_count, g.frame_size, g.total_size);

    /* 试试 mmap 零拷贝访问 */
    char *base = mmap(NULL, g.total_size, PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0);
    if (base == MAP_FAILED) { perror("mmap"); return 1; }

    /* 确保模拟 DMA 在跑（若驱动 USE_FAKE_IRQ=1 会默认启动；这里可强制一下） */
    int on = 1; ioctl(fd, IOCTL_RUN_FAKE, &on);

    /* 使用 poll 等待“有帧可读” */
    struct pollfd pfd = { .fd = fd, .events = POLLIN };
    for (int it = 0; it < 10; ++it) {
        int r = poll(&pfd, 1, 3000);
        if (r == 0) { puts("timeout"); continue; }
        if (r < 0) { perror("poll"); break; }

        /* 读取 head/tail，按环逐帧消费（零拷贝） */
        struct ring_index idx;
        if (ioctl(fd, IOCTL_GET_INDEX, &idx) < 0) { perror("ioctl idx"); break; }

        while (idx.tail != idx.head) {
            char *frame = base + (size_t)idx.tail * g.frame_size;
            printf("[frame @%u] %.60s", idx.tail, frame);  /* 打印前 60 字节 */
            /* 消费：用 read() 推进 tail（也可定义新的 ioctl 来推进 tail） */
            char tmp[4096];
            ssize_t n = read(fd, tmp, sizeof(tmp)); /* 内核里推进 tail */
            if (n < 0) { perror("read"); break; }
            idx.tail = ring_next(idx.tail, g.frame_count);
        }
    }

    munmap(base, g.total_size);
    close(fd);
    return 0;
}
```

## 3.34_关键点与工程建议

- **中断线程化**：`request_threaded_irq(top, thread, IRQF_*, ...)`
   顶半部只做最小工作（ack/屏蔽），**数据搬运放线程化 bottom-half**，可睡眠、可拿 mutex，更易维护。
- **索引并发**：
  - `head` 在 **IRQ/bottom-half** 更新；`tail` 在 **read/ioctl** 更新；
  - 用 `spin_lock_irqsave()` 保护二者（head 在 IRQ ctx 可能被访问）。
- **丢帧策略**：环满时，本示例选择**丢最旧帧**（推进 tail）。你也可在 head 停住并统计溢出，由用户尾追。
- **用户态无锁协议**：
  - `mmap` 后，用户用 `IOCTL_GET_INDEX` 取 head/tail 的**一致快照**，逐帧读取，最后调用 `read()` 或自定义 `IOCTL_POP_FRAME` 推进 tail。
  - 若要**完全零拷贝**且不愿 `read()`，可定义 `IOCTL_CONSUME(n)` 后端推进 tail。
- **真实硬件嵌入**：
  - 把 `produce_one_frame()` 调用放到 `my_irq_thread()` 中，依据 HW 的完成队列循环推进；
  - 如果是 **多描述符 DMA**，可把每个完成的 buf 映射到环上对应帧；
  - DMA 引擎需要**预提交**新的描述符（refill），保证持续流水。
- **安全与权限**：`mmap` 映射的是可写共享区，必要时限制 `PROT_WRITE` 或拆分读写队列。
- **扩展**：
  - 多实例：参照我们前面“multi-instance”模板，开 `DEV_COUNT>1`，每实例一套环；
  - `read_iter`/`write_iter`：换成 iter 风格，友好 `splice`/`sendfile`；
  - **dma-buf**：跨设备/进程共享（如 GPU/Camera pipeline）时考虑；
  - **中断计数/统计**：`debugfs` 暴露统计项，便于压测。

## 3.35_接口速查(新增/变化)

| 接口/类型              | 头文件               | 作用                                          |
| ---------------------- | -------------------- | --------------------------------------------- |
| `IOCTL_CONSUME`        | `<linux/ioctl.h>`    | 消费 1 帧（推进 `tail`）                      |
| `IOCTL_CONSUME_N`      | `<linux/ioctl.h>`    | 批量消费 N 帧                                 |
| `READ_ONCE/WRITE_ONCE` | `<linux/compiler.h>` | 读取/写入共享变量的原子语义（避免编译器重排） |

并发原则：

- **生产者**（IRQ 线程化 bottom-half / hrtimer 回调）推进 `head`；
- **消费者**（用户态+ioctl）推进 `tail`；
- 访问 `head/tail` 时用 `spin_lock_irqsave()`（因为 head 在 IRQ ctx 更新）。

## 3.36_内核驱动(零拷贝消费版)_mydma_irq_ring_zc.c

> 说明：在上一版基础上改动点很集中：删除 `read()` 推进 tail 的语义；新增 `IOCTL_CONSUME`/`IOCTL_CONSUME_N`。其余路径（DMA coherent、mmap、poll/epoll、hrtimer 模拟“中断”）不变。

```c
// SPDX-License-Identifier: GPL-2.0
/*
 * mydma_irq_ring_zc.c
 * DMA + IRQ + ring + poll + mmap + ioctl-consume (zero-copy)
 * Linux 6.1 语义
 */
#include <linux/module.h>

#include <linux/device.h>

#include <linux/compiler.h>

#define CLASS_NAME   "mydmairq_zc"
#define DEV_COUNT    1

/* 帧环参数 */
#define FRAME_SIZE    4096
#define FRAME_COUNT   64
#define DMA_BYTES     (FRAME_SIZE * FRAME_COUNT)

/* 模拟 DMA 完成中断 */
#define USE_FAKE_IRQ  1
#define FAKE_IRQ_HZ   200

/* ---------- ioctl 定义 ---------- */
#define IOCTL_MAGIC       'Z'
struct ring_geom { unsigned frame_size, frame_count, total_size; };
struct ring_index { unsigned head, tail; };

#define IOCTL_GET_GEOM    _IOR(IOCTL_MAGIC, 1, struct ring_geom)
#define IOCTL_GET_INDEX   _IOR(IOCTL_MAGIC, 2, struct ring_index)
#define IOCTL_CLR_INDEX   _IO(IOCTL_MAGIC,  3)
#define IOCTL_RUN_FAKE    _IOW(IOCTL_MAGIC, 4, int)
/* ★ 新增：零拷贝消费接口 */
#define IOCTL_CONSUME     _IO(IOCTL_MAGIC,  5)                 /* 消费1帧 */
#define IOCTL_CONSUME_N   _IOW(IOCTL_MAGIC, 6, unsigned)       /* 消费N帧 */

struct mydev {
    dev_t            devno;
    struct cdev      cdev;
    struct class    *cls;
    struct device   *dev;
    struct device   *parent;

    void            *cpu_addr;     /* coherent CPU addr */
    dma_addr_t       dma_handle;   /* DMA addr */
    size_t           bytes;

    unsigned         frame_size;
    unsigned         frame_count;
    unsigned         head;         /* 生产者推进 */
    unsigned         tail;         /* 消费者推进 */
    spinlock_t       idx_lock;

    wait_queue_head_t rdq;

#if USE_FAKE_IRQ
    struct hrtimer   timer;
    int              running;
    u64              seqno;
#endif
    int              irq;
};

/* ====== 全局设备（单实例） ====== */
static struct mydev g;

/* ====== Ring 工具 ====== */
static inline void *frame_ptr(struct mydev *d, unsigned idx)
{ return (char *)d->cpu_addr + (size_t)idx * d->frame_size; }

static inline unsigned ring_next(struct mydev *d, unsigned i)
{ return (i + 1) % d->frame_count; }

static inline bool ring_full(struct mydev *d, unsigned head, unsigned tail)
{ return ring_next(d, head) == tail; }

static inline bool ring_empty(struct mydev *d, unsigned head, unsigned tail)
{ return head == tail; }

/* ====== 生产一帧（底半部或模拟中断调用） ====== */
static void produce_one_frame(struct mydev *d)
{
    unsigned long flags;
    unsigned head, tail;

    spin_lock_irqsave(&d->idx_lock, flags);
    head = d->head; tail = d->tail;
    if (ring_full(d, head, tail)) {
        /* 溢出策略：丢弃最旧一帧 */
        d->tail = ring_next(d, tail);
    }
    head = d->head;
    spin_unlock_irqrestore(&d->idx_lock, flags);

    /* 填充帧（演示：写入序号与时间戳） */
    {
        char *p = frame_ptr(d, head);
        memset(p, 0, d->frame_size);
#if USE_FAKE_IRQ
        snprintf(p, 64, "frame#%llu ts=%llu ns\n",
                 (unsigned long long)READ_ONCE(d->seqno)++,
                 (unsigned long long)ktime_get_ns());
#else
        snprintf(p, 64, "hw_frame ts=%llu ns\n",
                 (unsigned long long)ktime_get_ns());
#endif
    }

    /* 推进 head 并唤醒读者 */
    spin_lock_irqsave(&d->idx_lock, flags);
    d->head = ring_next(d, d->head);
    spin_unlock_irqrestore(&d->idx_lock, flags);

    wake_up_interruptible(&d->rdq);
}

/* ====== 中断骨架（可选） ====== */
static irqreturn_t my_irq_top(int irq, void *dev_id)
{
    return IRQ_WAKE_THREAD;
}
static irqreturn_t my_irq_thread(int irq, void *dev_id)
{
    produce_one_frame(dev_id);
    return IRQ_HANDLED;
}

/* ====== hrtimer 模拟“DMA 完成” ====== */
#if USE_FAKE_IRQ
static enum hrtimer_restart my_timer_cb(struct hrtimer *t)
{
    struct mydev *d = container_of(t, struct mydev, timer);
    if (READ_ONCE(d->running)) {
        produce_one_frame(d);
        hrtimer_forward_now(&d->timer, ktime_set(0, NSEC_PER_SEC/FAKE_IRQ_HZ));
        return HRTIMER_RESTART;
    }
    return HRTIMER_NORESTART;
}
#endif

/* ====== file_operations ====== */

static int my_open(struct inode *inode, struct file *filp)
{
    filp->private_data = &g;
    try_module_get(THIS_MODULE);
    return 0;
}
static int my_release(struct inode *inode, struct file *filp)
{
    module_put(THIS_MODULE);
    return 0;
}

/* ★ 不再提供 read() 消费 tail —— 全部由 ioctl 完成零拷贝消费 */

/* poll：有帧可读则返回 POLLIN */
static __poll_t my_poll(struct file *f, struct poll_table_struct *wait)
{
    struct mydev *d = f->private_data;
    unsigned long flags;
    __poll_t mask = 0;
    poll_wait(f, &d->rdq, wait);
    spin_lock_irqsave(&d->idx_lock, flags);
    if (!ring_empty(d, d->head, d->tail))
        mask |= POLLIN | POLLRDNORM;
    spin_unlock_irqrestore(&d->idx_lock, flags);
    return mask | POLLOUT | POLLWRNORM; /* 写路径非关键，这里标记可写 */
}

/* mmap：映射整个环给用户态 */
static int my_mmap(struct file *f, struct vm_area_struct *vma)
{
    struct mydev *d = f->private_data;
    size_t len = vma->vm_end - vma->vm_start;
    if (len > d->bytes) return -EINVAL;
    return dma_mmap_coherent(d->parent, vma, d->cpu_addr, d->dma_handle, d->bytes);
}

/* ioctl：查询/控制 + ★零拷贝消费 */
static long my_ioctl(struct file *f, unsigned int cmd, unsigned long arg)
{
    struct mydev *d = f->private_data;
    unsigned long flags;

    switch (cmd) {
    case IOCTL_GET_GEOM: {
        struct ring_geom g = {
            .frame_size  = d->frame_size,
            .frame_count = d->frame_count,
            .total_size  = d->bytes,
        };
        if (copy_to_user((void __user *)arg, &g, sizeof(g))) return -EFAULT;
        return 0;
    }
    case IOCTL_GET_INDEX: {
        struct ring_index i;
        spin_lock_irqsave(&d->idx_lock, flags);
        i.head = d->head; i.tail = d->tail;
        spin_unlock_irqrestore(&d->idx_lock, flags);
        if (copy_to_user((void __user *)arg, &i, sizeof(i))) return -EFAULT;
        return 0;
    }
    case IOCTL_CLR_INDEX: {
        spin_lock_irqsave(&d->idx_lock, flags);
        d->head = d->tail = 0;
        spin_unlock_irqrestore(&d->idx_lock, flags);
        return 0;
    }
    case IOCTL_CONSUME: {
        /* 消费1帧：若空则 -ENODATA，若有则推进 tail */
        spin_lock_irqsave(&d->idx_lock, flags);
        if (ring_empty(d, d->head, d->tail)) {
            spin_unlock_irqrestore(&d->idx_lock, flags);
            return -ENODATA;
        }
        d->tail = ring_next(d, d->tail);
        spin_unlock_irqrestore(&d->idx_lock, flags);
        return 0;
    }
    case IOCTL_CONSUME_N: {
        unsigned n, i = 0;
        if (copy_from_user(&n, (void __user *)arg, sizeof(n))) return -EFAULT;
        if (!n) return 0;
        spin_lock_irqsave(&d->idx_lock, flags);
        while (i < n && !ring_empty(d, d->head, d->tail)) {
            d->tail = ring_next(d, d->tail);
            ++i;
        }
        spin_unlock_irqrestore(&d->idx_lock, flags);
        /* 返回“实际消费多少”更友好：把 i 写回 arg */
        if (copy_to_user((void __user *)arg, &i, sizeof(i))) return -EFAULT;
        return 0;
    }
    case IOCTL_RUN_FAKE: {
#if USE_FAKE_IRQ
        int on;
        if (copy_from_user(&on, (void __user *)arg, sizeof(on))) return -EFAULT;
        if (on) {
            if (!READ_ONCE(d->running)) {
                WRITE_ONCE(d->running, 1);
                hrtimer_start(&d->timer, ktime_set(0, NSEC_PER_SEC/FAKE_IRQ_HZ), HRTIMER_MODE_REL);
            }
        } else {
            WRITE_ONCE(d->running, 0);
            hrtimer_cancel(&d->timer);
        }
        return 0;
#else
        return -ENOTTY;
#endif
    }
    default:
        return -EINVAL;
    }
}

static const struct file_operations fops = {
    .owner          = THIS_MODULE,
    .open           = my_open,
    .release        = my_release,
    .poll           = my_poll,
    .mmap           = my_mmap,
    .unlocked_ioctl = my_ioctl,
    .llseek         = no_llseek,
};

/* ====== 装卸 ====== */
static int __init my_init(void)
{
    int ret;

    ret = alloc_chrdev_region(&g.devno, 0, DEV_COUNT, DEV_NAME);
    if (ret) return ret;

    cdev_init(&g.cdev, &fops);
    g.cdev.owner = THIS_MODULE;
    ret = cdev_add(&g.cdev, g.devno, DEV_COUNT);
    if (ret) goto err_chr;

    g.cls = class_create(THIS_MODULE, CLASS_NAME);
    if (IS_ERR(g.cls)) { ret = PTR_ERR(g.cls); goto err_cdev; }

    g.dev = device_create(g.cls, NULL, g.devno, NULL, DEV_NAME "0");
    if (IS_ERR(g.dev)) { ret = PTR_ERR(g.dev); goto err_cls; }
    g.parent = g.dev;  /* 教学：用 /sys/class/... 的 device 作为 DMA 设备 */

    g.bytes       = DMA_BYTES;
    g.frame_size  = FRAME_SIZE;
    g.frame_count = FRAME_COUNT;

    g.cpu_addr = dma_alloc_coherent(g.parent, g.bytes, &g.dma_handle, GFP_KERNEL);
    if (!g.cpu_addr) { ret = -ENOMEM; goto err_dev; }
    memset(g.cpu_addr, 0, g.bytes);

    spin_lock_init(&g.idx_lock);
    init_waitqueue_head(&g.rdq);
    g.head = g.tail = 0;

#if USE_FAKE_IRQ
    hrtimer_init(&g.timer, CLOCK_MONOTONIC, HRTIMER_MODE_REL);
    g.timer.function = my_timer_cb;
    g.seqno = 0;
    WRITE_ONCE(g.running, 1);
    hrtimer_start(&g.timer, ktime_set(0, NSEC_PER_SEC/FAKE_IRQ_HZ), HRTIMER_MODE_REL);
#endif

    pr_info("%s: loaded. major=%d, frames=%u x %uB = %zuB\n",
            DEV_NAME, MAJOR(g.devno), g.frame_count, g.frame_size, g.bytes);
    return 0;

err_dev:
    device_destroy(g.cls, g.devno);
err_cls:
    class_destroy(g.cls);
err_cdev:
    cdev_del(&g.cdev);
err_chr:
    unregister_chrdev_region(g.devno, DEV_COUNT);
    return ret;
}

static void __exit my_exit(void)
{
#if USE_FAKE_IRQ
    WRITE_ONCE(g.running, 0);
    hrtimer_cancel(&g.timer);
#endif
    dma_free_coherent(g.parent, g.bytes, g.cpu_addr, g.dma_handle);
    device_destroy(g.cls, g.devno);
    class_destroy(g.cls);
    cdev_del(&g.cdev);
    unregister_chrdev_region(g.devno, DEV_COUNT);
    pr_info("%s: unloaded\n", DEV_NAME);
}

module_init(my_init);
module_exit(my_exit);
```

## 3.37_用户态(零拷贝消费者)_zc_consumer.c

- 初始化：`IOCTL_GET_GEOM` 获取几何，`mmap` 整个环；
- 事件循环：`poll` 等待可读 → `IOCTL_GET_INDEX` 拿一致快照 → 按 `tail..head-1` 遍历帧 → **直接用 `base + idx\*frame_size` 处理** → 调用 `IOCTL_CONSUME_N` 批量推进 `tail`。
- 全程**无数据拷贝**（只在内核/用户共享区中处理）。

```c
// gcc -O2 -Wall -o zc_consumer zc_consumer.c
#include <stdio.h>
#include <stdlib.h>
#include <fcntl.h>
#include <sys/ioctl.h>
#include <sys/mman.h>
#include <sys/poll.h>
#include <unistd.h>
#include <string.h>
#include <errno.h>

#define DEV_PATH "/dev/mydmairq_zc0"

#define IOCTL_MAGIC       'Z'
struct ring_geom { unsigned frame_size, frame_count, total_size; };
struct ring_index { unsigned head, tail; };
#define IOCTL_GET_GEOM    _IOR(IOCTL_MAGIC, 1, struct ring_geom)
#define IOCTL_GET_INDEX   _IOR(IOCTL_MAGIC, 2, struct ring_index)
#define IOCTL_CLR_INDEX   _IO(IOCTL_MAGIC,  3)
#define IOCTL_RUN_FAKE    _IOW(IOCTL_MAGIC, 4, int)
#define IOCTL_CONSUME     _IO(IOCTL_MAGIC,  5)
#define IOCTL_CONSUME_N   _IOW(IOCTL_MAGIC, 6, unsigned)

static inline unsigned next_u(unsigned i, unsigned n) { return (i + 1) % n; }

int main(void)
{
    int fd = open(DEV_PATH, O_RDWR);
    if (fd < 0) { perror("open"); return 1; }

    /* geometry */
    struct ring_geom g;
    if (ioctl(fd, IOCTL_GET_GEOM, &g) < 0) { perror("IOCTL_GET_GEOM"); return 1; }
    printf("geom: %u frames x %uB = %uB\n", g.frame_count, g.frame_size, g.total_size);

    /* mmap ring */
    char *base = mmap(NULL, g.total_size, PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0);
    if (base == MAP_FAILED) { perror("mmap"); return 1; }

    /* 确保模拟 DMA 在跑 */
    int on = 1; ioctl(fd, IOCTL_RUN_FAKE, &on);

    struct pollfd pfd = { .fd = fd, .events = POLLIN };
    for (int it = 0; it < 10; ++it) {
        int r = poll(&pfd, 1, 3000);
        if (r == 0) { puts("poll: timeout"); continue; }
        if (r < 0) { perror("poll"); break; }

        /* 拿索引快照 */
        struct ring_index idx;
        if (ioctl(fd, IOCTL_GET_INDEX, &idx) < 0) { perror("GET_INDEX"); break; }

        /* 遍历 tail..head-1，处理帧（这里演示打印前 60 字节） */
        unsigned t = idx.tail, consumed = 0;
        while (t != idx.head) {
            char *frame = base + (size_t)t * g.frame_size;
            printf("[consume %u] %.60s", t, frame);
            t = next_u(t, g.frame_count);
            ++consumed;
        }

        if (consumed) {
            /* 批量标记消费完成 —— 零拷贝推进 tail */
            unsigned n = consumed;
            if (ioctl(fd, IOCTL_CONSUME_N, &n) < 0) { perror("CONSUME_N"); break; }
            /* 内核会把“实际推进了多少”写回 n；这里我们不强依赖 */
        }
    }

    munmap(base, g.total_size);
    close(fd);
    return 0;
}
```

## 3.38_工程细节与建议

- **零拷贝协议**：
  1. 生产者更新帧内容 → 推进 `head` → `wake_up_interruptible(&rdq)`；
  2. 用户态 `poll` 返回 → `GET_INDEX` 快照 → 直接处理 `mmap` 区数据；
  3. 处理完成后 `CONSUME_N` 推进 `tail`（无需 `read()`）。
- **一致性与可见性**：
  - 使用 **coherent DMA**（`dma_alloc_coherent`）保证 CPU/设备一致性；
  - 对 `head/tail` 使用 `spin_lock_irqsave` + `READ_ONCE/WRITE_ONCE`，避免重排。
- **溢出策略**：
  - 当前策略：环满时**丢最旧帧**（推进 `tail`）；
  - 可切换为“停住生产者 + 统计丢帧”，由用户加快消费。
- **多进程/多线程**：
  - 若允许多个消费者，需要更复杂的“引用计数/位图”标记；
  - 本模板默认单消费者，逻辑更清晰。
- **真实硬件接入**：
  - 在 `my_irq_thread()` 中按**完成队列**循环 `produce_one_frame()`；
  - 记得 **refill** DMA 描述符，保持流水不断；
  - 若平台 non-coherent，但你使用 coherent API，平台会处理一致性；否则请改用 streaming + `dma_sync_*`（不推荐 mmap 到用户态）。
