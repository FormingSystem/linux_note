## 1.2_入门心智模型(不提_API_也能看懂)

- 现实痛点 → Input 要解决什么（噪声/越界/半帧/多消费者/策略归属/功耗）
- 四角色与边界：硬件（产样本）→ 驱动（采样成帧+声明边界）→ input core（钳位/抑抖/分发）→ 用户态（手势/空间策略）
- 端到端时序：IRQ/轮询 → 采样 → 成帧 → 分发 → 用户态读取（帧语义）

## 1.5_两条完整流水线(问题_to_作用_to_定位_to_细节)

- **LCD 电容触摸屏（I²C，多点，DIRECT）**
  - 噪声/越界/同帧一致/多消费者 → 用 **`min/max/fuzz/res` + 帧语义** 解决
  - 用户态做手势与坐标变换
- **ADC 双轴摇杆**
  - 机械回中抖、死区 → **`fuzz` 抑抖** + **`flat` 建议死区（用户态采纳）**

## 1.6_同步与并发语义(必须讲透)

- **采集可睡、上报不睡**：IRQ 上下文、线程化 IRQ、workqueue、进程上下文
- 设备级顺序：input core 短自旋锁；处理层环形缓冲 + 唤醒
- 帧一致性与 TOCTOU：帧号或“前后检查 data-ready + 有限重读”
- 关机/休眠顺序：`disable_irq_sync()` → 改电源/状态；唤醒源配置
- 背压与丢帧提示：用户态自愈重同步流程
