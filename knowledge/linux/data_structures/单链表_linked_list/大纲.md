# 第1章_Linux内核链表

本文是 Linux 内核链表专题的统一入口。原有长篇内容已按概念、操作、并发、调试和应用拆分到当前目录，其他文档应统一链接到本大纲。

需要注意，目录名中的“单链表”是历史命名。本专题主要讲解 Linux 内核的 `struct list_head`：它是侵入式双向循环链表，并非通常意义上只包含 `next` 指针的单链表。正文统一使用“Linux 内核链表”或“`list_head` 链表”表述，涉及真正的单向链表时会单独说明。

## 1.1_学习目标

完成本专题后，应能够：

- 解释 `struct list_head` 为什么只保存连接关系，不直接保存业务数据。
- 使用 `container_of()` 从链表节点取得宿主对象。
- 正确完成链表初始化、插入、删除、遍历和判空。
- 说明 `READ_ONCE()`、`WRITE_ONCE()` 与链表并发安全之间的边界。
- 设计初始化、错误回滚、退出清理和调试验证流程。
- 根据访问频率、并发方式和查找复杂度判断是否应改用哈希表、红黑树或其他容器。
