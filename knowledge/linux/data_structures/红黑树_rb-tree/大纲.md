# 第1章_红黑树学习路线大纲

这组笔记的主线是：

```text
树基础
	↓
二叉树
	↓
BST
	↓
BST 退化与平衡动机
	↓
旋转
	↓
2-3-4 树
	↓
红黑树结构语义
	↓
Linux 内核 rbtree 工程实现
	↓
B 树 / B+ 树扩展
	↓
Maple Tree 与 VMA 源码分析
```

当前章节已经把红黑树理论、Linux 6.12 `rbtree` 源码实现、cached / augmented / RCU / 验证场景基本串完。后续不再单独新增“红黑树插入修复”“红黑树删除修复”的纯理论章节，因为这些内容已经分别被第 6、7、10、11 章覆盖。

## 1.2_以后写作边界

第 13 章只做扩展和对照，不再重复前面已经讲过的内容。

已经由前文覆盖的内容：

```text
树、二叉树、BST 基础：第 1-3 章
平衡动机和旋转：第 4-5 章
2-3-4 树的查找、插入、删除：第 6 章
红黑树性质、黑高、插入 / 删除修复目标：第 7 章
Linux rbtree 的工程实现：第 8-12 章
Maple Tree 和 VMA 场景：第 6.9、12.7 节
```

第 13 章只保留四个重点：

```text
1. 为什么从红黑树继续看 B 树 / B+ 树；
2. B 树和 B+ 树的结构差异；
3. 为什么 B+ 树适合页级索引和范围查询；
4. 红黑树、2-3-4 树、B 树、B+ 树在工程场景里的选择边界。
```

第 14 章是一个工程澄清章节，重点不是重新实现 Maple Tree，而是说清楚：

```text
Maple Tree 主要替代的是 VMA 管理中的 rbtree / linked list / vmacache；
它不是整个内核里所有 rbtree 的替代品；
它也不是公平调度从 CFS 走向 EEVDF 的原因。
```

第 15 章开始进入 Linux 6.12 Maple Tree 源码，但只做源码地图和 API 分层，不把 `mas_store()`、节点分裂、gap 更新、VMA 修改路径全部塞进同一章。

第 15 章保留六个重点：

```text
1. Maple Tree 相关源码文件各自负责什么；
2. struct maple_tree、struct maple_node、struct ma_state 的结构语义；
3. mtree_*()、mt_*()、mas_*()、vma_iter_*() 的分层关系；
4. mm_struct.mm_mt 和 vma_iterator 如何接入 VMA 管理；
5. vma_lookup()、find_vma()、find_vma_intersection() 的语义差异；
6. 后续继续拆查找、写入、gap search、mmap / munmap / mprotect 源码的章节边界。
```
