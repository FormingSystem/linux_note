# 13.2 命令速查表

------

## 1. 基础构建命令

| 命令                 | 作用                           | 备注                                     |
| -------------------- | ------------------------------ | ---------------------------------------- |
| `make menuconfig`    | 打开配置界面                   | 类似内核配置，保存后生成 `.config`       |
| `make <defconfig>`   | 使用指定 defconfig 初始化配置  | 例如 `make qemu_arm_versatile_defconfig` |
| `make savedefconfig` | 保存当前配置为最小化 defconfig | 输出到 `defconfig` 文件                  |
| `make`               | 执行完整构建                   | 会编译工具链、bootloader、kernel、rootfs |

------

## 2. 清理相关命令

| 命令                  | 作用                             | 使用场景           |
| --------------------- | -------------------------------- | ------------------ |
| `make clean`          | 清理目标文件，但保留下载与工具链 | 改动小，可快速重构 |
| `make distclean`      | 清理所有构建文件，包括工具链     | 相当于回到初始状态 |
| `make <pkg>-clean`    | 清理某个包的目标文件             | 改动单一包时使用   |
| `make <pkg>-dirclean` | 清理包的所有源码与构建           | 强制重新下载+构建  |

------

## 3. 包管理与调试

| 命令                      | 作用                      | 备注                        |
| ------------------------- | ------------------------- | --------------------------- |
| `make <pkg>`              | 单独构建某个包            | 例如 `make busybox`         |
| `make <pkg>-rebuild`      | 重新构建包（clean+build） | 常用于调试补丁              |
| `make <pkg>-menuconfig`   | 打开包的配置界面          | 支持 Linux、U-Boot、BusyBox |
| `make <pkg>-patch`        | 提取包源码并应用补丁      | 输出到 `output/build/<pkg>` |
| `make <pkg>-show-depends` | 显示包依赖                | 辅助分析依赖关系            |

------

## 4. 文件系统与镜像

| 命令                   | 作用                   | 说明                  |
| ---------------------- | ---------------------- | --------------------- |
| `make list-defconfigs` | 列出所有可用 defconfig | 便于查找平台          |
| `make graph-depends`   | 生成依赖图（PDF）      | 需安装 `graphviz`     |
| `make legal-info`      | 生成许可证信息         | 用于合规性分析        |
| `make sdk`             | 生成交叉编译 SDK       | 输出到 `output/host/` |

------

## 5. 输出目录结构

| 路径             | 内容                                              |
| ---------------- | ------------------------------------------------- |
| `output/build/`  | 各个包的源码与构建目录                            |
| `output/host/`   | 构建好的交叉工具链与 host 工具                    |
| `output/target/` | rootfs 的最终目录结构                             |
| `output/images/` | 生成的镜像文件（zImage、rootfs.ext4、sdcard.img） |

------

## 6. 实用技巧

1. **加速构建**

   ```bash
   make -j$(nproc)
   ```

2. **查看命令详情**（调试模式）

   ```bash
   make V=1
   ```

3. **强制重新生成镜像**

   ```bash
   make target-finalize
   make target-rootfs
   ```

# 13.3 常见问题 FAQ

------

## 1. 构建阶段

### Q1: 为什么 `make` 时下载失败？

**原因**：默认使用国外源，网络不稳定。
 **解决方法**：

- 配置代理：

  ```bash
  export http_proxy=http://127.0.0.1:7890
  export https_proxy=http://127.0.0.1:7890
  ```

- 使用国内镜像（如清华/中科大源）；

- 预先下载 tarball 到 `dl/` 目录。

------

### Q2: 为什么提示缺少 `host-xxx` 包？

**原因**：宿主机缺少依赖。
 **解决方法**：

- 参考 Buildroot 手册的依赖列表；

- 在 Ubuntu/Debian 下：

  ```bash
  sudo apt-get install build-essential git bc bison flex libncurses5-dev
  ```

------

### Q3: 构建时提示 “compiler not found”？

**原因**：交叉编译工具链未正确构建或路径错误。
 **解决方法**：

- 确认 `output/host/bin/` 下是否有 `arm-buildroot-linux-gnueabihf-gcc`；

- 如果缺失，执行：

  ```bash
  make toolchain-dirclean
  make toolchain
  ```

------

## 2. 镜像生成

### Q4: rootfs 镜像太小，编译出错？

**原因**：`BR2_TARGET_ROOTFS_EXT2_SIZE` 设置不足。
 **解决方法**：

- 在 `menuconfig → Filesystem images → ext2/3/4 root filesystem image` 调整大小；
- 或修改 `genimage.cfg` 中的分区大小。

------

### Q5: 烧录后 U-Boot 找不到内核？

**原因**：镜像偏移或分区配置错误。
 **解决方法**：

- 确认 `genimage.cfg` 与 U-Boot `bootcmd` 一致；
- 使用 `fdisk -l sdcard.img` 查看分区布局。

------

## 3. 启动阶段

### Q6: Kernel panic: "No init found"？

**原因**：rootfs 中没有 `/sbin/init`。
 **解决方法**：

- 确认 Buildroot 中启用了 **BusyBox init**；
- 临时调试：在 bootargs 添加 `init=/bin/sh`。

------

### Q7: 启动卡住，没有串口输出？

**原因**：可能是 U-Boot 未写入正确偏移。
 **解决方法**：

- 确认烧录命令：

  ```bash
  dd if=u-boot-spl.bin of=/dev/sdX bs=1K seek=1
  ```

- 确认波特率与板子默认配置一致（常见 115200）。

------

## 4. 用户空间

### Q8: 程序运行提示缺少动态库？

**原因**：依赖库未包含在 rootfs。
 **解决方法**：

- 使用 `ldd ./myapp` 查看依赖；
- 在 Buildroot `menuconfig` 中启用对应库。

------

### Q9: 常用命令（如 `ifconfig`、`ping`）不存在？

**原因**：BusyBox 配置过于精简。
 **解决方法**：

```bash
make busybox-menuconfig
```

启用相关命令后，重新构建 BusyBox。

------

### Q10: 如何在 rootfs 中添加额外文件？

**方法**：

- 使用 overlay：

  ```
  board/myboard/rootfs_overlay/etc/myconfig.conf
  ```

- 或 post-build 脚本：

  ```bash
  echo "mydata" > $TARGET_DIR/etc/myconfig.conf
  ```

------

## 5. 工程管理

### Q11: 为什么我的修改丢失了？

**原因**：直接改 `.config` 或 Buildroot 源码。
 **解决方法**：

- 使用 `defconfig` 保存配置；
- 项目定制放在 **External Tree**。

------

### Q12: 如何保证构建可复现？

**方法**：

- 固定 Buildroot 版本（推荐 LTS，如 2022.02）；
- 使用 `savedefconfig` 管理配置；
- 使用 Git 管理 external tree。

------

## 小结

- 构建失败，多数是 **依赖或下载问题**；
- 启动失败，多数是 **分区或 init 问题**；
- 用户空间问题，多数是 **库或 BusyBox 配置缺失**；
- 工程维护问题，多数是 **配置未保存或修改不当**。
