## 第一篇 · 可见性与顺序（Visibility & Ordering）

> 仅讲“道”，不引环境

### 第1章　内存可见性：READ/WRITE_ONCE 与发布-获取

- 〔概念〕CPU 视角“看得见/看不见”、happens-before
- 〔能做/不能做〕能保证跨 CPU 的状态可见；不负责设备 I/O 顺序
- 〔核心用法模式〕标志发布/指针发布最小模式
- 〔混搭与边界〕与原子/自旋/RCU 的关系
- 〔常见坑〕把 `barrier()` 当屏障；只用 ONCE 管复合不变量
- 〔最小模板〕“标志+数据”的发布-获取片段
- **表格**：
  - 〔概念区分表〕CPU↔CPU 可见性元素对照
  - 〔用法速览表〕READ/WRITE_ONCE + smp_* 场景速查
  - 〔核对表〕跨 CPU 标志/指针检查项

### 第2章　I/O 顺序：`*_relaxed` vs 非 relaxed 与 `mb/rmb/wmb`

- 〔概念〕CPU 程序顺序 ≠ 设备观测顺序
- 〔核心用法模式〕配置→**确认点**→启动；读序选择
- 〔混搭与边界〕与 DMA 门铃、与自旋锁顺序的区别
- 〔常见坑〕“锁=I/O 有序”的误解；滥用 `mb()`
- 〔最小模板〕顺序化寄存器写封装
- **表格**：
  - 〔概念区分表〕CPU↔CPU vs CPU↔设备 顺序差异
  - 〔用法速览表〕`*_relaxed/writel` 与 `wmb/rmb/mb` 选用
  - 〔核对表〕所有“配置→启动/门铃”是否显式顺序化

## 第二篇 · 互斥与临界区（Mutual Exclusion）

### 第3章　自旋锁（不可睡一侧）

- 〔概念〕短临界区，获取/释放自带内存序
- 〔核心用法模式〕`spin_lock_irqsave` 保护 ring 指针；锁外唤醒
- 〔混搭与边界〕与 mutex 的锁序（mutex 外、spin 内）；与 waitqueue 条件评估
- 〔常见坑〕自旋区内可睡/长操作
- 〔最小模板〕IRQ→环形队列→唤醒
- **表格**：
  - 〔概念区分表〕自旋 vs 互斥 能力/限制
  - 〔用法速览表〕自旋锁在各上下文可/不可
  - 〔核对表〕自旋临界区瘦身与唤醒位置

### 第4章　互斥锁 / 读写信号量（可睡一侧）

- 〔概念〕控制面、慢路径
- 〔核心用法模式〕ioctl/sysfs 与数据面分离；`rwsem` 的读多写少
- 〔混搭与边界〕与自旋的嵌套顺序；与线程化中断/工作队列
- 〔常见坑〕把数据面包进 mutex；锁反转
- 〔最小模板〕配置通道骨架
- **表格**：
  - 〔概念区分表〕mutex vs rwsem 取舍
  - 〔用法速览表〕可睡/不可睡上下文对照
  - 〔核对表〕锁序一致性检查

------

## 第三篇 · 读多写少与快照（Read-Mostly Structures）

### 第5章　seqcount/seqlock（读重试快照）

- 〔概念〕写入递增代次；读侧重试
- 〔核心用法模式〕`read_seqbegin/retry` + `write_seqbegin/end`
- 〔混搭与边界〕与 RCU 的选择边界
- 〔常见坑〕读侧可睡；写段过大
- 〔最小模板〕统计/时间戳快照
- **表格**：
  - 〔概念区分表〕seqcount vs seqlock
  - 〔用法速览表〕适用场景/读侧限制
  - 〔核对表〕写段长度/读重试项

### 第6章　RCU（读无锁，写延迟回收）

- 〔概念〕发布/读取/宽限期回收
- 〔核心用法模式〕`assign_pointer/dereference` + `call_rcu/synchronize_rcu`
- 〔混搭与边界〕与 kref 的保活；与等待队列/自旋
- 〔常见坑〕离开读侧继续用旧指针；回收过早
- 〔最小模板〕设备表查找 + kref
- **表格**：
  - 〔概念区分表〕RCU vs seqcount 选择矩阵
  - 〔用法速览表〕RCU 发布/回收 API 放置点
  - 〔核对表〕宽限期/引用协作项

## 第四篇 · 事件与等待（Events & Waiting）

### 第7章　等待队列（waitqueue）

- 〔概念〕条件为真就绪（电平语义）
- 〔核心用法模式〕“锁内条件 + 醒后重检”；锁外搬运
- 〔混搭与边界〕与自旋/RCU/seqcount 条件构造；与 poll/epoll
- 〔常见坑〕条件锁外评估；唤醒风暴
- 〔最小模板〕阻塞读骨架
- **表格**：
  - 〔概念区分表〕电平 vs 边沿 语义
  - 〔用法速览表〕wait_event* 变体选用
  - 〔核对表〕条件评估/重检/唤醒点

### 第8章　完成量（completion）

- 〔概念〕一次性交互（边沿语义）
- 〔核心用法模式〕`reinit_completion` / `complete` / `wait_for_completion*`
- 〔混搭与边界〕与线程化中断/工作队列；与 PM 上电握手
- 〔常见坑〕未 `reinit` 复用；不可睡上下文等待
- 〔最小模板〕启动握手
- **表格**：
  - 〔概念区分表〕等待队列 vs 完成量
  - 〔用法速览表〕一次性 vs 持续流 选型
  - 〔核对表〕一次性语义与重置

## 第五篇 · 执行路径与上下文（Execution Context）

### 第9章　中断/软中断/线程化中断/工作队列

- 〔概念〕不可睡→可睡的任务迁移
- 〔核心用法模式〕顶半部最小化；线程化/工作承接重活
- 〔混搭与边界〕与 waitqueue/completion、与 mutex 放置
- 〔常见坑〕取消不同步导致“晚到回调”
- 〔最小模板〕顶半部→线程/工作流水线
- **表格**：
  - 〔概念区分表〕各上下文能力/限制
  - 〔用法速览表〕取消/同步 API 选用
  - 〔核对表〕顶半部不做可睡操作；全部 `_sync` 取消

### 第10章　定时器/高精度定时器（timer/hrtimer）

- 〔概念〕时间驱动回调（不可睡）
- 〔核心用法模式〕`mod/del_timer_sync`、`hrtimer_start/cancel`
- 〔混搭与边界〕回调→工作队列传递；与 completion 超时配合
- 〔常见坑〕未同步取消
- 〔最小模板〕重传/防抖
- **表格**：
  - 〔概念区分表〕timer vs hrtimer
  - 〔用法速览表〕使用/取消语义
  - 〔核对表〕不可睡约束与取消顺序
