## 7.2 模块实现：自定义哈希增、删、查、改

我们将定义一个简单的结构体 `struct my_device`，并使用内核哈希表对其进行索引。

### 7.2.1 定义数据结构与初始化

首先，包含必要的头文件，并声明哈希表。

```c
#include <linux/module.h>
#include <linux/kernel.h>
#include <linux/hashtable.h> // 核心头文件
#include <linux/slab.h>      // kzalloc, kfree

// 1. 定义存储对象
struct my_data {
    int id;                  // 查找的 Key
    char name[20];
    struct hlist_node node;  // 必须包含这个节点，用于挂载到哈希表
};

// 2. 声明哈希表：指定名字和桶的大小（2^3 = 8个桶）
// 内核会自动处理 hash_32 运算逻辑
DEFINE_HASHTABLE(my_hash_table, 3); 
```

### 7.2.2 核心操作：增、删、查

```c
// 增加数据
void add_data(int id, const char *name) {
    struct my_data *new_node = kzalloc(sizeof(*new_node), GFP_KERNEL);
    if (!new_node)
        return;

    new_node->id = id;
    strncpy(new_node->name, name, 20);

    // 将对象添加到哈希表，第三个参数是计算 hash 的 key
    hash_add(my_hash_table, &new_node->node, new_node->id);
    pr_info("HashModule: Added ID %d, Name %s\n", id, name);
}

// 查找数据
struct my_data* find_data(int id) {
    struct my_data *cur;
    // 使用 hash_for_each_possible 遍历特定桶，性能极高
    hash_for_each_possible(my_hash_table, cur, node, id) {
        if (cur->id == id)
            return cur;
    }
    return NULL;
}

// 删除数据并释放内存
void delete_data(int id) {
    struct my_data *obj = find_data(id);
    if (obj) {
        hash_del(&obj->node); // 从哈希表中移除
        kfree(obj);           // 物理释放
        pr_info("HashModule: Deleted ID %d\n", id);
    }
}
```

## 7.3 内存审计：安全退出与防泄漏处理

内核模块最忌讳的是卸载后留下“内存垃圾”。在删除整个哈希表时，必须使用带有 `_safe` 后缀的宏。

### 7.3.1 模块退出逻辑

```c
static void __exit my_hash_exit(void) {
    struct my_data *cur;
    struct hlist_node *tmp;
    int bkt;

    // 安全遍历并删除所有节点
    // bkt 是桶的迭代变量，tmp 是为了防止在删除当前节点时指针失效
    hash_for_each_safe(my_hash_table, bkt, tmp, cur, node) {
        pr_info("HashModule: Cleaning up ID %d\n", cur->id);
        hash_del(&cur->node);
        kfree(cur);
    }
    pr_info("HashModule: Exited\n");
}
```

## 7.5 关键注意事项（避坑指南）

> [!CAUTION]
>
> **1. 并发安全**：上述代码在单线程初始化时是安全的。但在多核环境下，如果存在并发写（`hash_add`/`hash_del`），你**必须**手动包裹一个 `spinlock_t`。
>
> **2. 桶的大小**：`DEFINE_HASHTABLE` 的第二个参数是 $2^n$ 的 $n$。如果你的数据量预计有 1000 个，建议设为 $8$（即 $2^8 = 256$ 个桶），以保持链表长度在合理范围。
>
> **3. GFP_KERNEL**：在 `kzalloc` 时，由于我们是在模块初始化或普通进程上下文中，使用 `GFP_KERNEL`。如果在中断上下文中，必须改用 `GFP_ATOMIC`。
