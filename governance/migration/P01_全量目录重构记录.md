# 第1章_全量目录重构记录

本次目录重构依据[仓库信息架构设计](../architecture/repository_information_architecture.md)执行，目标是建立知识本体、工程应用、平台差异、实践验证、研究证据、导航编排和仓库治理的稳定边界。

## 1.3_链接维护

迁移全部使用 `git mv`。链接工具根据 Git 重命名关系更新 Markdown、图片和 Obsidian 配置路径，并在写入后执行第二次扫描验证幂等性。

本次迁移后的检查目标：

```text

待重命名文件：0
可更新链接：0
歧义链接：0

Obsidian配置：0
```

## 1.4_后续治理

目录归属完成后，知识内容继续按以下任务演进：

1. 按治理需要逐步补充 Front Matter，不批量改写人工确认的正文结构。
2. 将理论数据结构与 Linux 数据结构实现进一步拆分。
3. 将通用机制正文从领域应用文档中去重。
4. 建立 `atlas/tracks` 学习路线和反向索引。
5. 建立 `publications/manifests` 出版物编排。
6. 保持正文阅读序号，并在出版构建中校正跨文档连续性。
7. 增加孤立文档、依赖关系和源码版本检查。
